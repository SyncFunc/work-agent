# Step M6.5 端到端验收（resume 跨重启 + fork 跨重启 + CI 绿 + 零回归）

## 实现方案
- **目标**：以真实（fixture 录制）流程验证「跨重启恢复」与「fork 跨重启」端到端可用；CI 全绿、覆盖率报告产出、回归零。
- **改动文件**：
  - 新增 `tests/e2e/test_recovery_e2e.py`：启动 daemon → 跑一个简单任务（RecordedModel 或 fixture）→ 读 `session_id` → 杀 daemon → 重启 → `resume <id>` → 断言 messages 重建、可继续。
  - 新增 `tests/e2e/test_fork_e2e.py`：fork 一个会话 → 重启 → 父与子均 `resume` 成功且子含父前缀、互不串台。
  - `README.md` / `AGENTS.md` 补充 `resume` / `fork` 用法。
- **关键流程**：见 M6.2 恢复 + M6.1 fork；本步只做**串联与断言**，不引入新持久化逻辑。
- **依赖/环境**：daemon 进程管理（subprocess 启停）；可用 `RecordedModel` 避免真实 LLM。

## 验收标准
- [ ] `resume` 跨 daemon 重启：恢复后 messages 与崩溃前一致，能继续完成任务（含中断场景注入「继续」）。
- [ ] `fork` 跨重启：父/子各自 `resume` 成功，子会话 events 含父前缀、独立分支、互不串台；`list_sessions` 显示 `parent_session_id` 血缘。
- [ ] CI（fast job）全绿：`pytest`(unit+integration) + ruff + basedpyright + coverage 报告产出。
- [ ] 全量回归零：新增 M6 代码后 `pytest` 总数增加且无既有用例失败。
- [ ] 不变量：跨重启恢复基于完整未压缩 `EventStream`；fork 复制语义在恢复路径无特殊分支。
