# Step M6.5 端到端验收（resume 跨重启 + fork 跨重启 + CI 绿 + 零回归）

## 实现方案
- **目标**：以真实（fixture 录制）流程验证「跨重启恢复」与「fork 跨重启」端到端可用；CI 全绿、覆盖率报告产出、回归零。
- **改动文件**：
  - 新增 `tests/e2e/test_recovery_e2e.py`：启动 daemon → 跑一个简单任务（RecordedModel 或 fixture）→ 读 `session_id` → 杀 daemon → 重启 → `resume <id>` → 断言 messages 重建、可继续。
  - 新增 `tests/e2e/test_fork_e2e.py`：fork 一个会话 → 重启 → 父与子均 `resume` 成功且子含父前缀、互不串台。
  - `README.md` / `AGENTS.md` 补充 `resume` / `fork` 用法。
- **关键流程**：见 M6.2 恢复 + M6.1 fork；本步只做**串联与断言**，不引入新持久化逻辑。
- **依赖/环境**：daemon 进程管理（subprocess 启停）；可用 `RecordedModel` 避免真实 LLM。

## 验收标准
- [x] `resume` 跨 daemon 重启：恢复后 messages 与崩溃前一致，能继续完成任务（含中断场景注入「继续」）。
- [x] `fork` 跨重启：父/子各自 `resume` 成功，子会话 events 含父前缀、独立分支、互不串台；`list_sessions` 显示 `parent_session_id` 血缘。
- [x] CI（fast job）全绿：`pytest`(unit+integration) + ruff + basedpyright + coverage 报告产出。
- [x] 全量回归零：新增 M6 代码后 `pytest` 总数增加且无既有用例失败。
- [x] 不变量：跨重启恢复基于完整未压缩 `EventStream`；fork 复制语义在恢复路径无特殊分支。

## 知识沉淀
- **端到端验证手段（两条 e2e，标记 `@pytest.mark.slow`）**：
  - `tests/e2e/test_recovery_e2e.py`：`test_resume_full_session_across_daemon_restart`（跑完→新建 `SessionStore(db)` + `Session.from_store` 模拟 daemon 重启→`messages` 与崩溃前一致→续跑成功）；`test_resume_interrupted_turn_across_daemon_restart`（人工写入「DECISION(tool_calls)+TOOL_USE 无 TOOL_RESULT」部分事件流模拟崩溃→重启→`detect_interruption` 为 True→重建 messages 注入「继续」user→续跑产出最终答案）。
  - `tests/e2e/test_fork_e2e.py`：`test_fork_across_daemon_restart_lineage`（fork→重启父子各自 `from_store`→子消息以父前缀开头；`list_sessions` 的 `parent_session_id` 血缘可见；父子各自续跑、互不串台——父续跑 FINAL 不出现在子流、子续跑 FINAL 不出现在父流）。
- **「daemon 重启」的忠实模拟**：daemon 重启 = 内存中 `SessionRegistry`/`Session` 全丢，仅 SQLite `SessionStore` 保留；故 e2e 用「新建 `SessionStore(同一 db)` + `Session.from_store`」复现冷启动（与 `start_daemon` 行为一致），无需起 OS 子进程，确定、快、不 flaky。
- **CI coverage**：`fast` job 末步 `pytest --cov=agent --cov-report=xml -q` 产出 `coverage.xml`；`[tool.coverage.run] source=["agent"]` + `omit=["tests/*","agent/testing/*"]`。
- **全量回归零验证**：M6.1→M6.5 累计新增 `pytest` 用例（含 e2e），`pytest -q` 全程 395 passed（含 1 个 e2e 基线首建 skip），无既有用例失败；`pytest -m slow` 4 passed。
- **M6 跨步骤结论（汇总）**：① 持久化以 `EventStream`(seq 单调、`transient` 不落盘) 为单一事实来源，永不压缩；② 恢复 = `SessionStore.load`→`rebuild_messages`（USER→user、DECISION→assistant(tool_calls)、TOOL_RESULT→tool、CLARIFY/PLAN→合成 tool 消息完成配对；中断则丢悬空 assistant + 注入「继续」）；③ `SessionStoreSink` 落盘须用会话级全局 seq 覆盖 per-run seq（坑）；④ fork = 复制父 events 前缀 + 记 `parent_session_id`，恢复无特殊分支；⑤ 测试金字塔 unit/integration/e2e 分层、默认跳过 slow；⑥ CI 快慢分离（ruff+basedpyright+pytest+cov 门禁，e2e 仅 nightly）。
