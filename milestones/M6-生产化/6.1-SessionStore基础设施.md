# Step M6.1 SessionStore 基础设施（含会话 fork）

## 实现方案
- **目标**：提供跨重启的会话持久化底座——sqlite 存储 `events` + `sessions` 元数据，事件写入通过 `EventStream` sink 零侵入接入；并原生支持**会话 fork**（从父会话事件前缀派生新分支会话）。
- **改动文件**：
  - 新增 `agent/context/session_store.py`（或 `agent/persistence/session_store.py`）：`SessionStore` + `SessionStoreSink`。
  - 改 `agent/core/session.py`：① `Session.__init__` 增加 `session_id` 入参（统一 daemon/CLI/persist 三处 id，修复当前 daemon 与 `Session` 各生成一份 sid 的不一致，见「关键决策」）；② 构造时 `stream.subscribe(SessionStoreSink(...))`。
  - 改 `agent/daemon/registry.py`：`SessionRegistry.new` 创建 handle 时落 `sessions` 行，把 `sid` 透传给 `Session`。
  - 改 `agent/cli.py`：`run`/`chat` 创建会话时同步落 `sessions` 行。

- **sqlite schema**（复用 `TraceStore` 的 sqlite 模式）：
  ```sql
  CREATE TABLE sessions (
    session_id TEXT PRIMARY KEY,
    name TEXT,
    parent_session_id TEXT,          -- ⭐ fork 血缘（NULL=根会话）
    created_at REAL, updated_at REAL,
    plan_mode INTEGER, plan_path TEXT,
    clarify_total INTEGER, root_span_id TEXT, model_meta_json TEXT
  );
  CREATE TABLE events (
    session_id TEXT, seq INTEGER, type TEXT, json TEXT,
    transient INTEGER, ts REAL,
    PRIMARY KEY (session_id, seq)
  );
  ```
- **关键决策（fork 设计）**：
  - **fork = 复制父 events 前缀到新会话**（而非 per-event `parentUuid` 链表）。理由：本项目 `EventStream` 是单调 `seq` 扁平序列，复制前缀实现「共享历史 + 独立分支」最简单、零侵入；事件规模完全可接受。Claude Code 用 `parentUuid` 链表避免复制，本项目选复制以求简单。
  - **`session_id` 统一**：当前 `SessionRegistry.new` 生成 `sid`，而 `Session.__init__` 又 `self.session_id = uuid.uuid4().hex`（两份不同 id）。M6.1 必须统一为单一 id（daemon/CLI 创建时生成，传入 `Session`），否则持久化 id 与运行时 id 对不上，恢复无从索引。
  - **写入时机**：仅在 `append`（非 `transient`）时落盘，`emit` 瞬时事件（如 `tool_call_delta`）不持久化，与 M7 回放缓冲一致。
- **依赖/环境**：Python 3.12 + `sqlite3`（标准库）；复用既有 `Event.to_json/from_json`；`basedpyright` 类型校验。

## 验收标准
- [x] `SessionStore.create` + `append_event` + `load` 往返：`load` 重建的 `EventStream` 与原始 `to_json` 内容逐条（seq/type/json）一致；`transient` 事件（如 `tool_call_delta`）不进 `events` 表。
- [x] `SessionStoreSink` 订阅 `EventStream` 后，每次 `append` 非 transient 事件即落盘，无需改 `loop` 代码（零侵入）。
- [x] `fork(parent)` 生成新 `session_id` 且其 `events` 为父会话前缀的完整拷贝；`list_sessions` 返回 `parent_session_id` 血缘。
- [x] `Session.__init__(session_id=...)` 生效，daemon/CLI 创建的 `Session` 与持久化行使用**同一 id**。
- [x] 不变量：`EventStream` 永不压缩；持久化事件即完整未压缩序列（呼应 M4 双轨）。

> 实施完成（2026-07-22）：
> - **模块位置**：`agent/context/session_store.py`（`SessionStore` + `SessionStoreSink`），复用 `TraceStore` 的 sqlite 模式。
