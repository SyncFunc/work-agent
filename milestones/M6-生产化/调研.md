## 一、现状全景：什么在内存、什么已落盘

M7 已完成 daemon 常驻 + WS + 多会话**内存态**切换回放，这是 M6 的前置基础。
但「跨进程 / 重启恢复」仍是空白——下图标出持久化边界。

```mermaid
flowchart TB
    subgraph MEM["❌ 仅内存（daemon 重启即丢）"]
        SR["SessionRegistry._sessions\ndict 会话索引"]
        EB["SessionHandle.event_buffer\ndeque(maxlen=200) 回放缓冲"]
        ES["EventStream._events\n对话事件序列（不落盘）"]
        MSG["Session.messages\n当前对话投影"]
    end
    subgraph DISK["✅ 已落盘（sqlite / 文件）"]
        TS["TraceStore\nspans / logs（M3.1）"]
        SM["SessionMemory\nsummary.md（M4.4）"]
        PS["PlanStore\nplan md+json（M1.4）"]
    end
    CLI["client --resume"] -->|"仅连 daemon 最近会话\n非存储恢复"| SR
    SR --> EB --> ES --> MSG
    note["⚠ 缺口：无统一的跨重启\n会话恢复（对话+事件流+计划+记忆）"]
```

**关键事实**
- `EventStream` 已实现 `to_json()` / `from_json()`（`M1.3`），但只在内存；`seq` 单调、`transient` 标记已就绪——这是落地持久化的天然接口。
- daemon 的 `SessionRegistry` 全内存，`resume` 只是连上 daemon 进程内的最近会话，daemon 没起 / 重启后**无法恢复**。
- sqlite 目前只有 `TraceStore` 与 `SessionMemory` 摘要，没有「整段会话」。

## 二、会话恢复（Session Recovery）外部方案

### 2.1 Claude Code — JSONL 仅追加 transcript ⭐ 最贴近本项目

```mermaid
flowchart LR
    A[".jsonl transcript\nappend-only · 每行自包含"] --> B["loadTranscript"]
    B --> C["parseJSONL\nMap&lt;UUID,msg&gt;"]
    C --> D["buildConversationChain\n沿 parentUuid 回溯反转(防环)"]
    D --> E{"中断检测\ndetectTurnInterruption"}
    E -->|"末条 tool_result\n未获响应"| F["interrupted_turn\n注入『继续』"]
    E -->|"末条 user 无 assistant"| G["interrupted_prompt"]
    E -->|"正常"| H["重建对话历史"]
    F --> H
    H --> I["recoverOrphanedParallelToolResults\nfilterUnresolvedToolUses"]
```

要点（与本项目映射）：
- 每会话一个 JSONL，`parentUuid → uuid` 链表支持 **fork 分支**与子 agent 侧链；压缩边界用 `null` parent 截断。
- 崩溃安全：append-only，忽略不完整末行；轻量恢复只读头尾各 64KB。
- **本项目映射**：`EventStream` 已是「单一事实来源 + `seq` 单调 + `to_json/from_json` + `transient`」，与 JSONL transcript **等价**。落地 = 把 `Event` 按 `seq` 落盘，`from_json` 重放 → 重建 `messages`。**零新概念、复用既有协议**。

### 2.2 OpenAI Agents SDK — `SQLiteSession`（透明自动持久化）

```mermaid
flowchart TB
    subgraph TURN["每个 Runner 轮次（透明）"]
        L["get_items(session_id)"] --> P["prepend 历史"] --> G["generate"] --> S["add_items 落盘"]
    end
    DB[("session_items\nsession_id, rowid")]
    S --> DB
    DB --> L
    SET["SessionSettings(limit)\n防上下文溢出"]
```

要点：借鉴「**透明自动**」API——`AgentLoop.step` 结束自动把本轮新增 events 落 sqlite，恢复时自动 `get_items` 重建，业务代码无感知。多 worker 场景再换 Redis/SQLAlchemy。

### 2.3 其他参考
- **Aider**：`.aider.chat.history` + `.aider.llm.history` 双文件。
- **OpenHands**：会话存后端（sqlite + 文件）。
- **agent-memory**（OctavianTocan）：本地 sqlite + 语义检索——属「长期记忆」，非「会话恢复」，可作 M6 之后增强。

## 三、测试金字塔（Agent Test Pyramid）外部方案

### 3.1 传统 70/20/10 对 Agent 失效

Tian Pan(2026-04) 核心论点：单元非真单元、集成烧钱、e2e 非确定。
改为「**成本 / 保真度**」三层金字塔：

```mermaid
flowchart TB
    subgraph T3["Tier 3 · 目标完成 eval（非确定·少·夜间/手动）"]
        E["真实任务多次跑\nLLM-as-a-Judge / promptfoo\n非单次 Pass/Fail"]
    end
    subgraph T2["Tier 2 · 工具交互 tool tapes（VCR 风格·中）"]
        I["录制响应→重放\n验证工具顺序/参数\n错误分支(超时退避/降级)"]
    end
    subgraph T1["Tier 1 · Prompt 契约（确定·零成本·每次 commit）"]
        P["prompt 快照 diff\ntool schema 校验\n解析器 / 输入清洗"]
    end
    T3 --- T2 --- T1
    note["❌ 否定：『注入 mock LLM 测真实决策』——模型即逻辑"]
```

### 3.2 业界共识（zylos / openhelm / szaher）
- **单元**：工具 + Mock LLM 响应（确定性、免费、底座最大）。
- **集成**：agent loop + 确定性 fixture，验证工作流。
- **e2e**：真实/录制任务，昂贵、评估向。

### 3.3 本项目现状与落点

```mermaid
flowchart LR
    subgraph NOW["现状（23 文件 / 380 passed）"]
        N1["组件单测（FakeModel）"]
        N2["少量集成 test_integration / daemon / client"]
        N3["❌ 无 unit/integration/e2e 分层"]
        N4["❌ 无 slow marker / ruff / cov"]
    end
    subgraph TARGET["目标分层"]
        U["tests/unit（扩 N1 + prompt 快照）"]
        IT["tests/integration（扩 N2 + EventStream 重放）"]
        E2["tests/e2e（慢·@mark.slow·夜间）"]
    end
    N1 --> U
    N2 --> IT
```

**落点**：本项目 `FakeModel`/`RecordingModel` 策略即 Tier1 核心，须扩为「契约 + fixture 重放」主力，**避免集成测试动辄调真实 LLM（烧钱 + 非确定）**。

## 四、CI 外部方案

业界通行：GitHub Actions + ruff + basedpyright + `pytest --cov` + Codecov。
agent 项目（OpenHands / SWE-bench）普遍**快慢分离**：

```mermaid
flowchart LR
    subgraph FAST["快 CI（push / PR 门禁）"]
        F1["checkout"] --> F2["setup-python 3.12"] --> F3["pip install -e .[dev]"]
        F3 --> F4["ruff check"] --> F5["basedpyright"] --> F6["pytest unit+integration"]
        F6 --> F7["pytest-cov --cov=agent --cov-report=xml"]
    end
    subgraph SLOW["慢 CI（nightly / manual / workflow_dispatch）"]
        S1["pytest e2e (--slow)\n真实 LLM eval / fixture 重放"]
    end
    FAST -.->|"不阻塞合并"| SLOW
```

## 五、结论与推荐

| 主题 | 推荐方案 | 依据 |
|---|---|---|
| 会话恢复存储 | **sqlite 主** + 可选 JSONL 导出 | 现有 `TraceStore` 基础设施可复用；JSONL 导出呼应 Claude Code |
| 持久化协议 | 复用 `EventStream` `to_json/from_json` | 已等价 JSONL transcript，零新概念 |
| 中断检测 | 末条 `tool_use` 无 `tool_result` → 注入「继续」 | 直接映射 Claude Code `interrupted_turn` |
| 测试分层 | unit / integration / e2e + slow marker | Tian Pan 成本/保真度金字塔 |
| 录制重放 | 用 `EventStream` 作天然 tape | 事件流即「工具交互录像带」 |
| CI | GitHub Actions 快慢分离 | OpenHands / SWE-bench 通行做法 |
