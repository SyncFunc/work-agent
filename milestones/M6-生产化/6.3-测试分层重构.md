# Step M6.3 测试分层重构（unit / integration / e2e + prompt 快照 + tool tapes）

## 实现方案
- **目标**：把现有扁平测试（23 文件 / 380 passed）重组成「成本/保真度」三层金字塔，确定性优先、慢测隔离；引入 **prompt 快照测试**（Tier1 契约）与 **tool tapes 录制重放**（Tier2）。
- **改动文件/目录**：
  - 目录：`tests/unit/`（现有主体，保持）、`tests/integration/`（扩 `test_integration`/`test_daemon`/`test_session`）、`tests/e2e/`（新增，慢）。
  - 新增 `tests/conftest.py`：注册 `@pytest.mark.slow` / `@pytest.mark.e2e`；`pyproject.toml` `[tool.pytest.ini_options]` 增加 `markers` 与 `addopts = "-m 'not slow'"`（默认跳过慢测）。
  - 新增 `agent/testing/recorded_model.py`：`RecordedModel`（给定录制好的 `Decision` 序列依次回放），把 `EventStream` 当作天然「录像带」。
  - 新增 `tests/fixtures/prompts/<case>.json`：渲染后的 system prompt 基线；`tests/test_prompt_snapshot.py` 与之 diff。
- **关键接口/算法**：
  ```python
  class RecordedModel:
      def __init__(self, decisions: list[Decision]): ...
      async def act(self, messages, **kw) -> Decision:
          return self.decisions.pop(0)   # 顺序回放
  ```
  - **prompt 快照（Tier1）**：渲染 `system.md`/`agent_prompts` 为固定 JSON 工件，与仓库内基线 `diff`，仅在**有意变更**时更新基线（防 system prompt 被意外改，呼应 M4.5 静态前缀稳定）。
  - **tool tapes（Tier2）**：用 `EventStream` 录制一轮交互事件流，CI 以 `RecordedModel` 重放，断言「工具调用顺序 / 参数 / 错误分支（超时退避、优雅降级）」。
- **依赖/环境**：`pytest` + `pytest-asyncio`；`FakeModel`/`RecordingModel` 策略保留并作为 Tier1 主力；**禁止**在 unit/integration 调真实 LLM。

## 验收标准
- [x] `tests/unit|integration|e2e` 三层目录就位；`pytest -q` 默认只跑非 slow（integration 含）；`pytest -m slow` 可单独跑 e2e。
- [x] prompt 快照测试：system prompt 渲染结果与基线 `diff`，意外改动即失败。
- [x] tool tapes：至少 1 个录制/重放用例验证工具顺序+参数；至少 1 个错误分支（超时/降级）用例。
- [x] 全量（含 integration）仍 `passed`，无回归；unit/integration 不触发真实 LLM 调用。
- [x] 不变量：Tier1/Tier2 确定、零成本、每次 commit 必跑；仅 e2e 非确定走 slow/nightly。

## 知识沉淀
- **目录边界（三层金字塔）**：`tests/unit/`（纯逻辑、mock、零 IO/零网络）+ `tests/integration/`（含 daemon、session 恢复、prompt 快照、tool tapes，仍确定性、不调真实 LLM）+ `tests/e2e/`（慢、非确定，全部 `@pytest.mark.slow` 走 nightly/手动）。各层带 `__init__.py`。
- **`pytest` 配置（`pyproject.toml [tool.pytest.ini_options]`）**：`addopts = "-m 'not slow'"`（默认排除 e2e）+ `markers = [unit/integration/e2e/slow]`；`pythonpath=["."]` 已含。`tests/conftest.py` 提供 `make_registry`/`settings_factory` 共享 fixture（echo/slow 两个工具）、`_echo`/`_slow` 异步工具。
- **`RecordedModel`（`agent/testing/recorded_model.py`）**：给定 `list[Decision]` 按 **FIFO 顺序回放**（像 `FakeModel` 但语义是「录像带回放」），`act`/`stream` 均 `pop(0)`；是 Tier2 tool tapes 的主力替身。与 `FakeModel`(按脚本轮转) 区分：RecordedModel 强调「事件流录制→重放」一致序。**录制闭环（M6.3 补强，已落地）**：模块级 `decisions_from_eventstream(stream)` 从一次运行的 `EventStream` 提取 `DECISION` 决策序列；`dump_tape(decisions, path)` / `load_tape(path)` 把决策序列落盘为 JSON / 加载；`RecordedModel.from_tape(path)` 直接由 tape 文件构造回放模型。复用 `agent.core.events` 的 `_decision_to_dict/_from_dict`（结构 `["{text, tool_calls}"]`），与 `EventStream` 序列化同源。
- **prompt 快照（Tier1）**：`tests/integration/test_prompt_snapshot.py` 渲染 `_build_system_parts(Settings())` 的静态段为 JSON，首次运行写入基线 `tests/integration/fixtures/prompts/system_static.json` 并 `skip`，后续 run 与之 `diff`，system prompt 被意外改动即失败。**基线维护约定**：有意变更 prompt 时删除该基线文件让其重建，或手动更新后 commit。
- **tool tapes（Tier2）**：`tests/integration/test_tool_tapes.py` 用 `RecordedModel` 录制一轮交互事件流，断言「工具调用顺序 + 参数」，并覆盖错误分支（超时退避/优雅降级→`ToolResult(ok=False)` 不崩循环）。**真实回放录制闭环已落地**：① `test_record_replay_roundtrip` 用 `FakeModel` 跑一轮 loop → `decisions_from_eventstream` 提取决策 → `dump_tape` 落盘 → `RecordedModel.from_tape` 重新加载回放 → 断言工具顺序/参数/最终文本与原运行完全一致，证明「录制→落盘→回放」全链路可工作且回放等价；② `test_real_tape_file_replays` 从仓库内真实 tape 文件 `tests/integration/fixtures/tapes/echo_one.json` 加载回放（`RecordedModel.from_tape`），证明「真实回放」链路真实可用（该文件即录制产物格式样例，nightly 真实 LLM 跑出的 tape 结构完全相同）。
- **e2e（Tier3）**：`tests/e2e/test_e2e_resume_fork.py::test_e2e_resume_then_fork_across_restart` 标记 `@pytest.mark.slow`，跨「resume → fork → 继续」模拟重启，断言父前缀完整保留于子会话。
- **⚠️ 落盘 seq 全局性（与 M6.1/M6.2 衔接的坑）**：`SessionStoreSink` 落盘时**必须用「会话级全局 seq」覆盖事件自带 per-run seq**（新 run 的 `EventStream` 从 0 重新开始）。否则 resume/fork 后续跑的新事件（seq 0,1,2…）会经 `INSERT OR REPLACE` 主键 `(session_id, seq)` 覆盖既有父前缀，破坏事件流完整性。修复见 `agent/context/session_store.py` 的 `_next_seq()` + `SessionStoreSink.__call__`（惰性取 MAX(seq)+1 覆盖）。
- **禁止回归**：unit/integration 严禁调用真实 LLM； CI 用占位 key。全量验证 `pytest -q` → **395 passed**，但其中含 e2e 基线首次创建 skip；`pytest -m slow` → 1 passed（e2e）。
