# Step M2.5 CLI 与 HITL 交互（AgentTransport.approve）

## 实现方案

- 目标：在统一协议 `AgentTransport` 加 `async approve(action)`，由 `TerminalTransport` 渲染审批面板。
- 改动文件：`agent/core/transport.py`、`agent/runtime/terminal_transport.py`。
- **M2 重构后无大变化**：仅去掉 deny 相关的 CLI 选项引用。
- 关键接口：

  ```python
  class AgentTransport(Protocol):
      ...
      async def approve(self, action: Action) -> bool:
          # 交互式：渲染「🔒 是否允许执行？」面板
          # 非交互：直接放行（gate.noninteractive_default 决定）
          ...
  ```

- **TerminalTransport.approve**：rich Panel + prompt_toolkit 异步确认。
- **CLI 选项**：`--approval-mode`（choices 四值：`on-request`/`unless-trusted`/`on-failure`/`never`）、`--sandbox-mode`、`--sandbox-profile`。

## 验收标准

- [x] 命令/测试：`tests/test_approval.py` 和 `tests/test_loop.py` 中 `_FakeTransport` 替代了旧的 `_FakeUI`：`test_transport_false_rejects`（transport 拒绝→不执行）、`test_transport_true_allows`（transport 批准→放行）、`test_unless_trused_asks_and_rejects`（集成测试：transport 拒绝→bash 被拦）。

## 知识沉淀

- **AgentTransport.approve** 替代了原有的 `ApprovalUI` Protocol：HITL 交互统一走 `AgentTransport`，不再有独立的 UI 回调类型。`ApprovalUI` 已删除。
- **`ApprovalGate` 不再持有 `ui`**：`authorize` 方法改为接收 `transport: AgentTransport | None` 参数，由 loop 在运行期传入。`ApprovalGate.__init__` 去掉了 `ui` 参数。
- **AgentTransport.approve** 必须 async（prompt_toolkit `prompt_async`）。
- **CLI 选项**：`--approval-mode` 选项值更新为 `on-request`/`unless-trusted`/`on-failure`/`never`。
