# Step M2.5 CLI 与 HITL 交互（AgentTransport.approve）

## 实现方案

- 目标：实现单步 HITL 的**人机交互层**——在统一协议 `AgentTransport` 加 `async approve(action)`，由 `TerminalTransport` 渲染「🔒 是否允许执行」面板并异步收集 y/n；`run`/`chat` 把 `transport`（已实现 `AgentTransport`）传给 `Session`（M1.6 重构后 `Session.step` 接收 `transport`，此处仅接入审批 CLI 选项），并在 `--approval-mode`/`--sandbox-*` CLI 选项接入配置。
- 改动文件：
  - `agent/core/transport.py`：`AgentTransport` Protocol 新增 `async def approve(self, action: Action) -> bool:`（设计文档 §3.3）。gate 经窄协议 `ApprovalUI` 消费（见 M2.2），`AgentTransport` 结构满足之。
  - `agent/runtime/terminal_transport.py`：`TerminalTransport.approve` 实现（rich Panel + prompt_toolkit 异步确认）；`run`/`chat` 增加 `--approval-mode` / `--sandbox-mode` / `--sandbox-profile` 选项，经 `load_settings(**overrides)` 注入。
  - 渲染：审批面板由 `TerminalTransport.approve` 内直接用 `console`/`Panel` 渲染（presenter 已并入 `AgentTransport`，无需额外 `on_approval` 回调）。
- 关键接口/算法（设计文档 §3.3 / §4.1）：

  ```python
  # agent/core/transport.py → AgentTransport (统一协议，已含 ask/confirm_plan/...)
  class AgentTransport(Protocol):
      @property
      def interactive(self) -> bool: ...
      async def ask(self, q) -> str: ...
      def show_questions(self, qs) -> None: ...
      def show_plan(self, plan) -> None: ...
      async def confirm_plan(self) -> bool: ...
      def notify(self, msg: str) -> None: ...
      def bind(self, stream) -> None: ...
      def close(self) -> None: ...
      def report_usage(self, usage, answer=None) -> None: ...
      async def approve(self, action: Action) -> bool: ...   # 新增：单步审批 HITL
  ```

  ```python
  # agent/runtime/terminal_transport.py → TerminalTransport.approve
  async def approve(self, action: Action) -> bool:
      if not self.interactive:
          return True   # 非交互由 gate.noninteractive_default 决定，这里不二次拦截
      self._console.print()  # 与流式文本空行分隔（沿用 M1.6 踩坑）
      self._console.print(Panel(
          f"[bold]{action.tool}[/]  ({action.risk})\n{action.description}",
          title="🔒 权限确认", border_style="yellow", expand=False))
      ans = await self._prompt_session.prompt_async("允许执行? [y/N] ")
      return ans.strip().lower() in ("y", "yes")
  ```

- **渲染**：`approve` 打印 `action.description`（工具名 + 风险 + 命令预览，限长防刷屏）；`y` 放行 / 其它拒绝。拒绝后 `loop` 侧已把 `ToolResult(ok=False,"rejected")` 落事件流（M2.4），无需 CLI 额外处理。
- **trace 记录**：`loop._exec_tools` 在 `gate.authorize` 前后各 emit `Event(type="approval", approved=bool, action=...)`（或复用现有 event 字段新增 `approval` 类型），便于 M5 审计/回放。沿用 `Event` 既有扩展方式（`type` 加 `"approval"`）。
- **CLI 选项**：`run`/`chat` 增加 `--approval-mode`（choices 四值，默认取 `settings.approval_mode`）、`--sandbox-mode`、`--sandbox-profile`；传入 `load_settings(approval_mode=..., sandbox_mode=..., sandbox_profile=...)`。选项仅覆盖配置，不硬编码默认。
- **与澄清/计划的 HITL 一致性**：`approve` 复用 `TerminalTransport` 的 `prompt_session`（prompt_toolkit 异步，已在 `ask`/`confirm_plan` 使用），**必须是 async**（前车之鉴 M1.6 踩坑⑥：同步 `prompt` 在已有事件循环里再 `asyncio.run()` 会崩）。
- 依赖/环境：沿用 rich + prompt_toolkit（M1.6 已引入）；无新依赖。

## 验收标准

- [ ] 命令/测试：`tests/test_cli.py` 扩展：
  - 注入 `FakeModel` 调 `bash` + `approval_mode=untrusted` + 假 `transport.approve` 返回 False → 退出码/结果体现拒绝、不真执行；

- [ ] 不变量：`AgentTransport.approve` 为 async；`Event.type` 新增 `"approval"` 不破坏既有事件解析（M1.3 `from_json` 兼容未知 type）；trace 含 approval 记录。

## 知识沉淀

> 完成本步后填写：AgentTransport.approve 签名、TerminalTransport 实现要点、CLI 选项、trace 事件。同步追加到 `knowledge/INDEX.md`。

- **AgentTransport 协议新增**：`async def approve(self, action: Action) -> bool`；所有 `AgentTransport` 实现（含测试假实现）必须补该方法（否则 Protocol 调用处类型/运行报错）。gate 经窄协议 `ApprovalUI` 消费（`AgentTransport` 结构满足）。
- **TerminalTransport.approve 铁律**：**必须 async**（prompt_toolkit `prompt_async`），不能同步 `prompt`（M1.6 踩坑⑥ 同因）；面板前 `console.print()` 空行与流式文本分隔（M1.6 踩坑）。
- **非交互语义**：`interactive=False` 时 `approve` 直接放行，真正决策权在 `gate.noninteractive_default`（默认 allow）——CLI 不在 `approve` 内二次拦截，避免与 gate 逻辑重复。
- **CLI 选项**：`--approval-mode`/`--sandbox-mode`/`--sandbox-profile` 经 `load_settings(**overrides)` 注入，优先级最高；不硬编码默认（默认取自 `Settings`）。
- **trace 事件**：`Event.type` 增 `"approval"`（`Event.approved: bool|None` / `Event.action: dict|None`），`from_json` 容错未知 type；供 M5 审计。
- **对 M2.6 约束**：本步的 HITL 行为是 `test_cli.py` / `test_loop.py` 集成审批用例的验收对象；假 `transport` 实现需含 `approve`。
