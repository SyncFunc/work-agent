# Step M2.6 测试与验收（test_sandbox / test_approval / 集成回归）

## 实现方案

- 目标：为 M2 全程补齐测试，确保沙箱、审批、集成三层都可自动化验证。
- 改动文件：`tests/test_sandbox.py`、`tests/test_approval.py`、`tests/test_loop.py`、`tests/test_cli.py`。
- **M2 重构后调整**：去掉 deny/escalated 相关用例；`UNTRUSTED`→`UNLESS_TRUSTED`；`allow`→`exec_policy`；提权测试不再需要 `enable_elevation=True`。
- 测试分层：

  **① 单元测试**
  - `test_sandbox.py`：执行器/工厂/Fake/External/Local 降级。
  - `test_approval.py`：四模式矩阵；`unless-trusted` + `exec_policy` 免审；`exec_policy` 在其它模式不生效；`on-request` 仅 `approval_request` 时 ASK；HITL 回调；规则匹配（前缀/正则/归一化）；`decide` 纯函数。

  **② 集成测试**
  - `test_loop.py`：`gate=UNLESS_TRUSTED` + 假 `ui` 拒 → bash 被拦；`gate=None` → M1 一致；提权：模型 `_approval_request` → ASK→批准 → 自动 `danger-full`；普通 ALLOW 不提权；`unless-trusted` + `exec_policy` 命中 → 免审。

  **③ 回归**
  - `test_tools.py`：`bash` 经 `get_executor()` 仍通过。
  - `test_plan.py`：**全绿**，PLAN 门控未被影响。

## 验收标准

- [ ] 命令/测试：`pytest -q` 全绿（M1 零回归）。
- [ ] 行为：集成用例能演示「unless-trusted + 拒 → 不执行」「提权自动生效」「exec_policy 免审」。
