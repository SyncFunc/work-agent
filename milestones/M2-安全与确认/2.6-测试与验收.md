# Step M2.6 测试与验收（test_sandbox / test_approval / 集成回归）

## 实现方案

- 目标：为 M2 全程补齐测试，确保沙箱、审批、集成三层都有可自动化验证；跑通全量 `pytest` 且既有 M1 测试零回归。
- 改动文件：新建 `tests/test_sandbox.py`、`tests/test_approval.py`；扩展 `tests/test_loop.py`（集成审批）、`tests/test_cli.py`（HITL 交互）、`tests/test_tools.py`（bash 走沙箱的回归）、`tests/test_plan.py`（PLAN 门控未受影响）。
- 测试分层（设计文档 §5 / 项目「测试驱动」原则）：

  **① 单元测试（确定性，mock LLM/root/网络）**
  - `test_sandbox.py`：`build_executor` 按 `mode` 返回正确实例；`FakeExecutor` 记录 `ExecRequest` 并脚本化返回；`ExternalExecutor` 直通执行 `echo`；`LocalExecutor` 在 CI Linux 跑通 `echo` 且隔离不可用时降级不报错（`profile=read-only` 时 `FakeExecutor` 断言 `ExecRequest.profile`）；枚举值合法。
  - `test_approval.py`：四模式 verdict 矩阵（设计文档 §4.3）；**deny 优先**于所有模式（含 `never`）；**allow 短路**；`noninteractive_default`（无 ui → 按默认）；注入假 `ui` 返回 True/False；规则前缀（`rm `）+ 正则（`/^curl .*x\.com/`）命中；`sudo rm x` 归一化后被 deny 命中；`read` 在 `untrusted` ALLOW、`exec` ASK；`on-request` 下 `approval_request=True` 的单条命令 → ASK（LLM 决定问此条），`approval_request=False` → ALLOW；`escalated=True` → 任何模式强制 ASK。

  **② 集成测试（FakeModel + 注入层）**
  - `test_loop.py` 扩展：`AgentLoop(model, registry, settings, sandbox=FakeExecutor(), gate=ApprovalGate(...))`：
    - `gate=UNTRUSTED` + 假 `ui` 拒 → `bash` 得 `ToolResult(ok=False, error 含 blocked/rejected)`，不真执行；
    - `gate=NEVER` + `deny=["rm "]` → `rm -rf x` DENY、`echo hi` 自动经 `FakeExecutor`；
    - `gate=None` → 与 M1 行为一致（直接执行）；
    - `on-failure` 模式：`bash` 执行失败（`res.ok is False`）→ 触发 `transport.approve` 问**补救方案**（重试/继续）。
  - `test_cli.py` 扩展：`run` 注入 `FakeModel` + 假 `transport`：`approval_mode=untrusted` + `transport.approve=False` 体现拒绝不执行；`--approval-mode never --sandbox-mode external` 覆盖生效；非交互不阻塞。

  **③ 回归（不变量守护）**
  - `test_tools.py`：`bash` 现有用例（`echo hi` 等）经 `get_executor()`（默认 `LocalExecutor` 或测试注入 `FakeExecutor`）仍通过，不依赖 root/网络。
  - `test_plan.py`：**全绿**，证明 PLAN 门控（`_risk_blocked`）未被 `ApprovalGate` 影响。
  - 全量 `pytest -q` 绿；`pytest --cov=agent` 沙箱/审批模块覆盖率达标。

- 依赖/环境：`pytest` + `pytest-asyncio`（M1 已有）；`FakeExecutor`/`FakeModel`/`RecordingModel` 复用；无新依赖。Windows CI 下 `LocalExecutor` 走 `CommandFilter` 应用层拦截（不打印告警），测试不依赖隔离可用。

## 验收标准

- [ ] 命令/测试：`pytest tests/test_sandbox.py tests/test_approval.py -q` 全绿；`pytest tests/test_loop.py tests/test_cli.py tests/test_tools.py tests/test_plan.py -q` 全绿；`pytest -q` 整体全绿（M1 零回归）。
- [ ] 行为：集成用例能演示「untrusted + 拒 → 不执行」「never + deny → 仍拦 rm」「PLAN 仍只放行 read」。
- [ ] 不变量：所有 LLM/root/网络均被 mock；`AgentLoop` 无 `gate`/`sandbox` 时行为 == M1（向后兼容）；`test_plan.py` 未被破坏（PLAN 与审批正交）。

## 知识沉淀

> 完成本步后填写：测试清单、覆盖率、回归守护点。同步追加到 `knowledge/INDEX.md`（并汇总 M2 里程碑级沉淀）。

- **测试清单**：`test_sandbox.py`（执行器/工厂/Fake/External/Local 降级）、`test_approval.py`（四模式/deny 优先/allow 短路/规则匹配/noninteractive）、`test_loop.py`+`test_cli.py`（集成审批 + HITL）、`test_tools.py`/`test_plan.py`（回归）。
- **回归守护铁律**：`gate=None`/`sandbox=None` 时 `AgentLoop` 行为必须与 M1 完全一致（向后兼容）；`test_plan.py` 必须保持绿（PLAN 门控与 `ApprovalGate` 正交，设计文档 §1）。
- **mock 纪律**：LLM 用 `FakeModel`/`RecordingModel`；执行用 `FakeExecutor`；HITL 用假 `transport.approve`；绝不真跑 root/网络/seccomp。
- **M2 里程碑级沉淀（汇总）**：
  - 接口：`SandboxProfile`(三值) / `ExecRequest` / `ExecResult` / `Executor` Protocol / `build_executor` / `get_executor`（注入点）；`ApprovalMode`(四值) / `Action` / `Decision` / `ApprovalGate.decide(纯函数)` / `async authorize`；`AgentTransport.approve(async)`（gate 经窄协议 `ApprovalUI` 消费）。
  - 决策顺序：`deny` > `escalated` > `read`非untrusted > `allow` > `mode`；**deny 永远优先**（安全不变量）；`escalated` 提权（装包/改系统配置）任何模式强制 ASK。
  - OS 降级：`local` 在 Linux landlock+seccomp(+unshare -n) / macOS Seatbelt / **原生 Windows 应用层 `CommandFilter` 主动拦截（不打印告警）**；Linux/macOS 旧内核降级为 ⚠️ 告警；`docker` 跨平台真隔离；`external` 直通。
  - 边界：`ApprovalGate` 仅 EXEC 模式；PLAN `_risk_blocked` 不动；拒绝/失败返回既有 `ToolResult(ok=False)` 落事件流、不崩循环。
  - 踩坑：HITL `approve` 必须 async（M1.6 踩坑⑥ 同因）；`bash` 经 `get_executor()` 不直连 subprocess；列表型配置（deny/allow）项目 YAML 整体覆盖默认（非追加）。
