# Step M2.3 分层权限配置（Settings）

## 实现方案

- 目标：把沙箱与审批的**所有可变策略**落进声明式 `Settings`（YAML + env + CLI）。
- 改动文件：`agent/config/settings.py`（`Settings` 增字段）。
- **M2 重构后精简**：去掉 `approval_deny`、去掉 `approval_elevate_on_approve`、`approval_allow` 改为 `approval_exec_policy`。
- 新增字段：

  ```python
  # agent/config/settings.py → class Settings
  sandbox_mode: str = "local"  # local | docker | external
  sandbox_profile: str = "workspace-write"  # read-only | workspace-write | danger-full
  approval_mode: str = "on-request"  # on-request | unless-trusted | on-failure | never
  approval_exec_policy: list[str] = []  # 执行策略（仅 unless-trusted 有效）
  noninteractive_default: str = "allow"  # 非交互无 HITL 时的默认 verdict
  elevated_sandbox_profile: str = "danger-full"  # 提权目标档位（ASK→批准且需联网时自动提升至此）
  ```

- **枚举校验**：`sandbox_mode`/`sandbox_profile`/`approval_mode` 用 pydantic `field_validator` 校验。
- **列表覆盖语义**：`approval_exec_policy` 是 pydantic `list`，项目级 YAML 整体覆盖默认值。
- **env 映射**：pydantic-settings 自动映射（如 `AGENT_SANDBOX_MODE` / `AGENT_APPROVAL_MODE`）。

## 验收标准

- [x] 命令/测试：`pytest` 已有 `tests/test_settings.py` 断言默认值正确（`test_m23_defaults_without_any_config`）；项目级 YAML 覆盖生效（`test_m23_yaml_overrides`）。
- [x] 行为：所有新字段有默认值（`test_m23_defaults_without_any_config`），旧配置/旧测试仍正常加载（`extra="ignore"` 静默忽略旧字段）。

## 知识沉淀

- **字段清单**：`sandbox_mode`/`sandbox_profile`/`approval_mode`/`approval_exec_policy`/`noninteractive_default`/`elevated_sandbox_profile`。
- **M2 重构去掉的字段**：`approval_deny`（不再需要）、`approval_elevate_on_approve`（提权默认自动）。
- **`approval_allow` 改为 `approval_exec_policy`**：语义更清晰——"哪些命令可以免审执行"。
- **env 命名**：pydantic-settings 默认映射 `AGENT_<FIELD>`。
