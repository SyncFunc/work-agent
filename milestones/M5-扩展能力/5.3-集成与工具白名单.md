# Step M5.3 集成与工具白名单

## 实现方案

**目标**：把 M5.1 的 `SkillLoader` 与 M5.2 的 `SubagentSpawner` 接入主 `AgentLoop`——① 把 Skill 触发目录注入系统提示；② 新增两个控制工具 `use_skill`（按需加载 skill 正文注入 conv）与 `spawn_subagent`（委派子任务、回填摘要）；③ 完成工具白名单/权限/沙箱向 subagent 的映射；④ fork 可选路径。

**改动文件**：
- `agent/core/control_tools.py`：新增 `USE_SKILL_TOOL` / `SPAWN_SUBAGENT_TOOL` 名常量 + schema；`collect_control_tools()` 按 settings 追加。
- `agent/core/loop.py`：`_exec_tools` 增加 `use_skill` / `spawn_subagent` 分支（在 `UnknownTool` 前，复用 tool_call_id 配对）；`_system_prompt()` 注入 skill 触发目录；`run()` 接收 `skill_loader` / `subagent_spawner` 可选依赖（构造或 run 注入）。
- `agent/core/session.py`：`Session.__init__` 构造 `SkillLoader` + `SubagentSpawner` 并传给 `AgentLoop`。
- `agent/config/settings.py`：新增 `Settings.skills`（`enabled`/`dirs`）+ `Settings.subagents`（`enabled`/`max_depth`/`auto_allow`）。
- `agent/core/prompts/system.md`：新增 `## Skills` 段渲染 `skills_catalog` 变量（由 loop 传入）。

**关键接口**：

```python
# agent/core/control_tools.py 新增
USE_SKILL_TOOL_NAME = "use_skill"
SPAWN_SUBAGENT_TOOL_NAME = "spawn_subagent"

USE_SKILL_TOOL = {
    "type": "function",
    "function": {
        "name": USE_SKILL_TOOL_NAME,
        "description": "调用一个已注册的 Skill（按需加载其正文到上下文）。参数 name。",
        "parameters": {"type": "object", "properties": {
            "name": {"type": "string", "description": "skill 名称（kebab-case）"},
            "args": {"type": "array", "items": {"type": "string"}, "description": "位置参数"},
        }, "required": ["name"]},
    },
}

SPAWN_SUBAGENT_TOOL = {
    "type": "function",
    "function": {
        "name": SPAWN_SUBAGENT_TOOL_NAME,
        "description": "把一个子任务委派给独立上下文的子 agent，返回其摘要。",
        "parameters": {"type": "object", "properties": {
            "agent": {"type": "string", "description": "agent 类型（explore/plan/general-purpose 或自定义名）"},
            "task": {"type": "string", "description": "委派给子 agent 的任务描述"},
        }, "required": ["agent", "task"]},
```

```python
# agent/core/loop.py 改动要点
class AgentLoop:
    def __init__(self, model, registry, settings, tracer=None, *,
                 plan_mode=None, plan_path=None, sandbox=None, gate=None,
                 skill_loader: "SkillLoader | None" = None,
                 subagent_spawner: "SubagentSpawner | None" = None):
        ...
        self.skill_loader = skill_loader
        self.subagent_spawner = subagent_spawner

    def _system_prompt(self, *, plan_mode=False, plan_path=None) -> str:
        catalog = ""
        if self.skill_loader is not None:
            catalog = self.skill_loader.catalog_prompt()
        return load_prompt("system").render(
            ..., skills_catalog=catalog)

    async def _exec_tools(self, calls, stream):
        for tc in calls:
            # 控制工具优先（在 UnknownTool 前）
            if tc.name == USE_SKILL_TOOL_NAME:
                return self._tool_use_skill(tc)        # 读正文→注入 conv（返回 ToolResult）
            if tc.name == SPAWN_SUBAGENT_TOOL_NAME:
                return await self._tool_spawn_subagent(tc, stream)
            ...
```

```python
# loop._tool_spawn_subagent 伪码
async def _tool_spawn_subagent(self, tc, stream):
    a = tc.arguments
    spec = self.subagent_spawner.get(a["agent"]) or BUILTIN_BY_NAME.get(a["agent"])
    if spec is None:
        return ToolResult(ok=False, error=f"unknown agent: {a['agent']}")
    # 当前 span 作为父 span
    parent = self._agent_span
    result = await self.subagent_spawner.spawn(
        spec, a["task"], depth=self._current_depth + 1, parent_span=parent,
        base_registry=self.registry, base_model=self.model)
    summary = f"[Subagent {spec.name}]\n{result.text}"
    return ToolResult(ok=True, output=summary)
```

**工具白名单映射**（复用 M5.2 `_subset_registry`）：`spawn_subagent` 调用 `SubagentSpawner.spawn` 时，`spec.tools`/`disallowed_tools` 决定子 `registry` 子集。`_exec_tools` 已 `registry.get` 配对，子 agent 不受影响。

**权限/沙箱映射**：`spec.permission_mode="plan"` → 子 `sandbox=build_executor('local', profile=read-only)` + `gate=ApprovalGate(mode="never")`（只读沙箱已兜底，不弹审批）。非交互子 agent 默认 `noninteractive_default="allow"`，避免卡 HITL。

**Skill 触发目录注入**：`SkillLoader.catalog_prompt()` 输出 `## 可用 Skills\n- name: trigger_text\n...`，注入 `system.md` 的 `## Skills` 段。模型看到描述后，用 `use_skill(name=...)` 触发——`loop._tool_use_skill` 读 `spec.render_body(args)` 注入 conv（作为一条 user 消息或 system 补充，建议追加到 `conv` 末尾供后续轮次使用）。

**fork 可选**：若 `spec.context=="fork"`（来自 skill 的 `context: fork`），`spawn` 时 `share_history=True`（拷贝父 `messages`）。

**依赖/环境**：
- M5.1 `SkillLoader` + M5.2 `SubagentSpawner` 已落地。
- `control_tools.collect_control_tools` 已支持 `plan_mode`/`has_plan` 条件；本步新增 `skills_enabled`/`subagents_enabled` 条件。
- `Settings` 新增 `skills`/`subagents` 嵌套配置（YAML 可覆盖）。

## 验收标准

- [ ] 系统提示包含 Skill 触发目录（仅 name + trigger_text，不含正文）。
- [ ] 模型调 `use_skill(name="x")` → 对应 skill 正文被读取、参数替换后注入 conv（断言 conv 新增一条含正文的消息）。
- [ ] 模型调 `spawn_subagent(agent="explore", task=...)` → 返回 `[Subagent explore]\n<摘要>` 的 `ToolResult`，且子 agent 用独立上下文运行（断言子 `messages` 从 `[]` 起）。
- [ ] 工具白名单：Explore subagent 调写类工具被拒（UnknownTool 降级或 registry 不含）。
- [ ] 权限映射：explore/plan subagent 在非交互下不弹审批（断言 `ApprovalGate` 未触发 `authorize`）。
- [ ] `settings.skills.enabled=False` 时 `use_skill` 不出现在工具列表；`subagents.enabled=False` 同理。
- [ ] 深度累加：嵌套 spawn 时 `depth+1` 传入；`depth>=max_depth` 抛 `RecursionError`。
- [ ] Trace 父子：子 agent span 的 `parent_id` 为父 `agent.run` span id。
- [ ] 不变量：主 agent 的 `EventStream` 不含子 agent 内部事件（仅最终的 tool_result 摘要）。
- [ ] 测试：`tests/test_loop.py` 追加集成用例（use_skill / spawn_subagent / 白名单 / 深度）；`tests/test_skills.py` / `tests/test_subagent.py` 已含单测。
