# Step M5.4 CLI 命令

## 实现方案

**目标**：为 Skill 与 Subagent 提供 CLI 交互入口——`/skills` 列出/查看已注册 Skill、`/agents` 列出已注册 Subagent 类型、交互式触发 Skill、后台运行 Subagent（可选）。复用 M1.6 的 `AgentTransport`/`TerminalTransport` 与 `Session`。

**改动文件**：
- `agent/cli.py`：`chat` REPL 增加 `/skills`、`/agents` 命令解析（复用 `/plan`/`/exec` 的「仅改 Session 状态、不调模型」模式）。
- `agent/core/transport.py`：`AgentTransport` 可选增 `show_skills(specs)` / `show_agents(specs)`（渲染列表面板）；`TerminalTransport` 实现 rich 面板。
- `agent/core/session.py`：`Session` 持有 `SkillLoader`/`SubagentSpawner`（M5.3 已建）；新增 `list_skills()`/`list_agents()` 便捷方法。
- `agent/skills/loader.py` / `agent/subagent.py`：各增 `summaries()` 返回精简列表（name + 一句话描述），供 CLI 展示。

**关键接口**：

```python
# agent/cli.py chat REPL 新增分支（在 /plan /exec 同类处理）
if user_input == "/skills":
    transport.show_skills(session.skill_loader.summaries())
    continue
if user_input == "/agents":
    transport.show_agents(session.subagent_spawner.summaries())
    continue
# 交互触发 skill（自然语言已由模型用 use_skill 处理；此处提供显式入口）
if user_input.startswith("/skill "):
    name = user_input[len("/skill "):].strip()
    spec = session.skill_loader.get(name)
    if spec is None:
        transport.notify(f"未找到 skill: {name}")
    else:
        # 直接把 skill 正文作为一条 user 消息注入下一轮（等价于模型调 use_skill）
        session.messages.append(Message(role="user",
            content=f"[Skill {name}]\n{spec.render_body()}"))
        transport.notify(f"已加载 skill: {name}")
    continue
```

**展示面板**（`TerminalTransport.show_skills`）：rich Table 列出 `name` / `description`(截断) / `paths`(若有) / 是否 `disable_model_invocation`。`/agents` 同理列出 `name` / `description` / `tools` / `model`。

**后台 Subagent（可选增强）**：M5.4 主路径是前台阻塞 `spawn_subagent`（模型调用即同步拿摘要）。后台运行可作为增强：`/agent <name> <task>` 启动一个 asyncio 任务，完成后再把摘要作为 user 消息回填并 notify。M5 先实现前台，后台留 `background` 字段接口。

**实时重扫**：会话中新增 `.agent/skills/` 或 `.agent/agents/` 文件后立即生效——`/skills` `/agents` 调用时重新 `discover()`（对齐 Claude 实时检测）。

**依赖/环境**：
- M5.3 已把 `SkillLoader`/`SubagentSpawner` 接入 `Session` 与 `AgentLoop`。
- `AgentTransport` 协议（M-refactor 统一传输层）。
- 不依赖真实 API；CLI 交互可用 `FakeModel` 注入测试（或仅展示列表，无需模型）。

## 验收标准

- [ ] `chat` 下输入 `/skills` 展示已注册 Skill 列表（name + 描述）。
- [ ] `/agents` 展示已注册 Subagent 类型（含内置 explore/plan/general-purpose）。
- [ ] `/skill <name>` 显式加载 skill 正文到下一轮 conv（断言 `session.messages` 新增一条 `[Skill ...]` 消息）。
- [ ] 未知 skill 名 → `notify` 提示未找到，不崩溃。
- [ ] 会话中新增 skill/agent 文件后，`/skills` `/agents` 重新发现（实时生效）。
- [ ] 不变量：CLI 命令不调模型（与 `/plan`/`/exec` 同模式，仅改 Session 状态）。
- [ ] 测试：`tests/test_cli.py` 追加 `/skills` `/agents` 展示用例（用 `FakeModel` + 临时 `.agent/skills` 目录）；`AgentTransport` 新增方法在 `_RecordingTransport` 补 no-op 以兼容旧测试。
