## 知识沉淀
> 完成本步后填写：接口签名、模块边界、约定、决策理由、踩坑。同步追加到 `knowledge/INDEX.md`。

- **模型边界（铁律）**：`Model` 只产出「决策」，绝不执行工具；工具执行在 M1.2 由 `ToolRegistry`/`AgentLoop` 负责。
- **核心接口签名**（位于 `agent/core/model.py`）：
  - `class Model(Protocol): async def act(self, messages) -> Decision` 且 `async def stream(self, messages) -> AsyncIterator[StreamEvent]`（流式增量输出）。
  - `StreamEvent(type, text=None, decision=None)`：`type="text"` 增量文本；`type="done"` 收尾回传完整 `Decision`（含流式聚合的工具调用）。
  - `Message(role, content=None, tool_calls=None, tool_call_id=None)`，`role ∈ {system,user,assistant,tool}`；tool 回执用 `tool_call_id` 关联。
  - `Decision(text=None, tool_calls=[])`；`is_final` 属性 = `not tool_calls`。
  - `ToolCall(id, name, arguments)`。
- **测试替身**：`FakeModel(script: list[Decision])` 按脚本依次返回并记录 `calls`；`RecordingModel(decision= | on_act=fn)` 记录 messages 并按需返回/回调生成。
- **生产模型（provider 无关）**：`OpenAICompatibleModel`（底层 OpenAI 兼容协议）。`OpenAICompatibleModel.from_settings(settings)` 离线构建（不联网，可注入假 client）；`create_model(settings)` 工厂返回默认实现。**代码无 DeepSeek 硬编码**——换 API 只改 `<项目根>/.agent/settings.yaml` 的 `llm.api_key`/`llm.base_url`/`llm.model`。`_to_openai/_from_openai` 做消息与工具调用互转（arguments 走 JSON）。`stream()` 增量产文本、结束回传聚合后的 `Decision`。
- **配置（位于 `agent/config/settings.py`）**：分层 YAML（用户级 + 项目级）+ CLI。优先级（高→低）：`CLI(init) > 项目级 YAML > 用户级 YAML > 内置默认`。**不读 `.env`/环境变量**。`load_settings(project_root=, **overrides)` 供 CLI/嵌入覆盖。默认 `llm.model="deepseek-v4-flash"`、`llm.base_url="https://api.deepseek.com"`、`max_iterations=25`。配置分层补遗见下方「配置分层（YAML）」。

### 配置分层（YAML，M1.1 完成后补充）
- 配置文件位置：项目级 `<project>/.agent/settings.yaml`（gitignore，优先级高）、用户级 `~/.agent/settings.yaml`（优先级低）；路径可被环境变量 `AGENT_PROJECT_ROOT` / `AGENT_USER_CONFIG_DIR` 覆盖（便于测试）。
- 实现：自定义 `YamlConfigSource(PydanticBaseSettingsSource)`，在 `__init__` 预加载「用户级→项目级」合并字典（项目级覆盖用户级），`get_field_value` 逐字段返回；`settings_customise_sources` 排序为 `init > YamlConfigSource`（合并用户级→项目级，项目级覆盖用户级）。
- YAML 支持扁平键（`llm_model: x`）或嵌套块（`llm: { model: x }`，加载时自动展平一层）。
- **密钥约定**：`llm.api_key` 写在项目级 `.agent/settings.yaml`（已被 gitignore，不会提交）。
- 模板：`agent/config/settings.example.yaml`；本项目真实配置：`.agent/settings.yaml`。
- **可观测占位（位于 `agent/obs/tracer.py`）**：`Tracer.span(name, kind, parent)` 返回上下文管理器；`span.set(**meta)` 打标；`Tracer.render()` 输出父子树。M5 再接 OTel/Langfuse。
- **测试约束**：`pyproject.toml` 设 `asyncio_mode="auto"`，异步测试无需 `@pytest.mark.asyncio`；用 `monkeypatch.setenv` 验证配置分层。
- **踩坑**：PowerShell 无 `tail`；模型名按用户要求为 `deepseek-v4-flash`（非 `deepseek-chat`）。
- **重构记录（用户反馈）**：① 去掉显式 DeepSeek 依赖——`DeepSeekModel` 重构为 provider 无关的 `OpenAICompatibleModel`，DeepSeek 仅作为默认配置值；② 补上流式输出——`Model` 协议新增 `stream()`，`StreamEvent` 分 `text`/`done`，工具调用在流式下按 `index` 增量聚合。
- **对 M2/M3 的约束**：`Decision.tool_calls` 形态即 M2 审批与 M3 压缩的输入；`Tracer` 的 `parent` 关系即 M5 父子 span 基础，新增调用务必显式传 parent。
