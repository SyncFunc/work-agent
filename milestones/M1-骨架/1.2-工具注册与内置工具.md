# Step M1.2 工具注册与内置工具

## 实现方案
- 目标：实现 `ToolRegistry`（`@tool` 装饰器注册，自描述 JSON Schema），落地 `bash`/`read`/`write` 三个内置工具，均带单测（用 temp 目录，不碰真实文件系统根）。
- 改动文件：
  - `agent/runtime/registry.py`（`ToolSpec(name, fn, risk, schema)`、`register`、`get`、`list`）
  - `tools/bash.py`、`tools/fs.py`（`read`/`write`）
  - `tests/test_registry.py`、`tests/test_tools.py`
- 关键接口：
  ```python
  def tool(name: str, risk: str = "read"): ...


  # ToolSpec.fn 签名: async def (args: dict) -> ToolResult
  ```
- 风险分级先预留字段（`read`/`edit`/`exec`），M2 才接审批。

## 验收标准
- [ ] `pytest tests/test_registry.py tests/test_tools.py` 全部通过。
- [ ] `ToolRegistry.list()` 返回 3 个工具及 schema；未知工具名 `get()` 抛 `UnknownTool`。
- [ ] `write` 写入临时目录后 `read` 能读回；`bash` 执行 `echo` 能拿到 stdout。

## 知识沉淀

- **接口签名**（`agent/runtime/registry.py`）：
  - `ToolSpec(name: str, fn: (dict)->Awaitable[ToolResult], risk: str="read", schema: dict)`；`to_openai()` 导出 OpenAI function tool 形态。
  - `tool(name, risk="read", schema=None)` 装饰器：套在 `async def (args: dict) -> ToolResult` 上，返回 **ToolSpec**（不直接注册）。
  - `ToolRegistry`：`register(spec)`、`get(name)->ToolSpec`(未知抛 `UnknownTool`)、`list()->list[ToolSpec]`、`run(name, args)->ToolResult`、`to_openai_tools()`。
  - `ToolResult(ok: bool, output: str="", error: str|None=None)`；`UnknownTool(KeyError)`。
  - `default_registry = ToolRegistry()` 全局单例；`tools/` 模块在**导入时**通过 `default_registry.register(...)` 完成登记（副作用写在 `tools/__init__.py`）。
- **模块边界**：`registry.py` 是纯确定性注册/调度，不含任何 IO；`tools/`（fs.py/bash.py）是内置工具实现，IO 在此处。Agent 循环（M1.3）只调用 `registry.run`，绝不自己执行。
- **风险分级**：`RISK_LEVELS=("read","edit","exec")`；`register` 校验非法 risk 抛 `ValueError`。`risk` 仅预留字段，M2 才接审批/沙箱。
- **测试约定**：用 `default_registry.run("name", args)` 调度；read/write 的工作根 = 进程 `cwd`，测试用 `monkeypatch.chdir(tmp_path)` 把根隔离到临时目录，且 `_resolve` 拒绝路径遍历（`path escapes root`）。
- **踩坑（重要，Windows/pytest）**：
  1. **pytest 发现子包**：新加 `agent/tools`、`agent/runtime` 等子包后，需 `pip install -e .` 刷新，并在 `pyproject.toml` 的 `[tool.pytest.ini_options]` 加 `pythonpath = ["."]`（否则 `No module named 'agent.tools'`）。
  2. **bash 超时在 Windows 上是硬问题**：`create_subprocess_shell` 经 `cmd.exe /c` 派生子进程，`proc.kill()` 只杀 `cmd.exe`，**孤儿子进程持管道直到自然退出**；且 ProactorEventLoop 下 `wait_for(communicate)` **无法取消管道读取**，超时只在进程退出后才生效。→ 解决方案：用 `asyncio.wait({communicate_task, asyncio.sleep(timeout)}, FIRST_COMPLETED)` 竞速；超时分支在 Windows 调 `taskkill /F /T /PID` 杀进程树（Unix 用 `proc.kill()`/`os.killpg`），再 `await wait_for(comm_task, 2)` 收尾。实测超时从 ~5s 降到 ~1.4s。
  3. **bash 吞掉超时**：超时返回 `ToolResult(ok=False, error="command timed out after Ns")`，**不向上抛异常**——测试应断言返回结果，而非 `pytest.raises(TimeoutError)`。
  4. 跨平台阻塞命令：Windows 无 `sleep` 命令，`bash` 超时测试用 `python -c "import time; time.sleep(5)"`（跨平台且必然阻塞）。
- **导出约定**：`agent/runtime/__init__.py` 重导出 `ToolRegistry/ToolResult/ToolSpec/UnknownTool/default_registry/tool/RISK_LEVELS`。
- **对 M2 的约束**：`risk` 字段即 M2 审批与沙箱的输入；`ToolResult.ok/error` 是事件流里 tool 回执的统一形态。
