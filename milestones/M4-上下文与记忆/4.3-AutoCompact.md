# Step M4.3 Auto Compact（9 段摘要 + Compact Boundary + 防漂移）

## 实现方案

**目标**：实现 Auto Compact 压缩器——当 Microcompact 后上下文仍超阈值时，把 `compact_boundary` 之前的历史整批压缩为 **9 段结构化摘要**（1 次模型调用），插入 Compact Boundary 标记，然后执行防漂移（重读最近文件 + 重注入固定底座）。

**关键接口**：

```python
# agent/context/compactors/auto_compact.py

COMPACT_SYSTEM_PROMPT = """你是一个上下文压缩助手。你的任务是把一段对话历史压缩成结构化摘要，保留关键信息，丢弃冗余细节。只输出文本，不要调用任何工具。输出格式：<summary>...</summary>。"""

COMPACT_USER_TEMPLATE = """请把以下对话历史压缩为 9 段结构化摘要：

1. **Primary Request and Intent** — 用户的原始请求和意图
2. **Key Technical Concepts** — 涉及的关键技术概念
3. **Files and Code Sections** — 涉及的文件路径和代码段
4. **Errors and Fixes** — 遇到的错误和修复方法
5. **Problem Solving** — 解决问题的思路和步骤
6. **All User Messages** — 所有用户消息的关键内容
7. **Pending Tasks** — 尚未完成的待办事项
8. **Current Work** — 当前正在进行的任务细节（包含原文引用）
9. **Optional Next Step** — 建议的下一步操作（包含原文引用）

对话历史：
{history_text}

请输出 <summary>...</summary> 格式的结果。第 8、9 段必须包含原文引用以防止语义漂移。"""


class AutoCompact:
    """Auto Compact 压缩器：1 次模型调用，生成 9 段结构化摘要。

    策略：
    - 提取 boundary 前的完整历史作为压缩源。
    - 发送压缩 prompt + 历史给 LLM（model.stream 或 model.act）。
    - 解析输出中 <summary>...</summary> 内容。
    - 用摘要替换 boundary 前历史，保留 boundary 后原文。
    - 连续失败 > 3 次则放弃（防止无限重试浪费 token）。
    """

    def __init__(self, model: Model, max_failures: int = 3):
        self.model = model
        self.max_failures = max_failures
        self.failure_count = 0

    async def compact(
        self,
        conv: list[Message],
        boundary: int,
    ) -> list[Message]:
        if boundary <= 0:
            return conv
        if self.failure_count > self.max_failures:
            return conv  # 放弃压缩

        history = conv[:boundary]
        history_text = self._format_history(history)

        prompt = COMPACT_USER_TEMPLATE.format(history_text=history_text)
        summary = await self._call_model(prompt)

        if summary is None:
            self.failure_count += 1
            return conv

        self.failure_count = 0
        # 用摘要替换 boundary 前历史
        return [
            Message(role="user", content=f"[Compact Summary]\n{summary}"),
        ] + conv[boundary:]

    async def _call_model(self, prompt: str) -> str | None:
        """调用模型获取摘要。返回 <summary> 标签内文本，失败返回 None。"""
        messages = [
            Message(role="system", content=COMPACT_SYSTEM_PROMPT),
            Message(role="user", content=prompt),
        ]
        try:
            decision = await self.model.act(messages)
            text = decision.text or ""
            if "<summary>" in text and "</summary>" in text:
                start = text.index("<summary>") + len("<summary>")
                end = text.index("</summary>")
                return text[start:end].strip()
            # 无标签时兜底取全文
            return text.strip() if text else None
        except Exception:
            return None

    def _format_history(self, messages: list[Message]) -> str:
        """把 messages 格式化为可读文本。"""
        lines = []
        for i, msg in enumerate(messages):
            role = msg.role
            content = msg.content or ""
            if msg.tool_calls:
                for tc in msg.tool_calls:
                    content += f"\n  → tool: {tc.name}({tc.arguments})"
            lines.append(f"[{i}][{role}] {content[:2000]}")
        return "\n".join(lines)
```

**防漂移机制**（集成到 ContextManager）：

```python
# agent/context/manager.py 新增
class ContextManager:
    def __init__(self, ...):
        ...
        self.auto_compact: AutoCompact | None = None
        self.recent_files: list[str] = []  # 最近操作的文件路径

    def track_file_access(self, path: str) -> None:
        """记录最近访问的文件（由 loop._exec_tools 回调，或 transport 事件驱动）。"""
        self.recent_files.append(path)
        if len(self.recent_files) > 10:
            self.recent_files = self.recent_files[-10:]

    async def compact(self) -> bool:
        """执行完整压缩流程：Microcompact → (若仍超) Auto Compact → 防漂移。"""
        # ① Microcompact（零成本）
        self.apply_microcompact()

        # ② 检查是否需要进一步压缩
        if not self.should_compact():
            return True

        # ③ Auto Compact（1 次调用）
        if self.auto_compact is not None:
            new_conv = await self.auto_compact.compact(self.conv, self.compact_boundary)
            if new_conv is not self.conv:
                self.conv = new_conv

        # ④ 标记边界
        self.mark_boundary()
        self.history.append(CompactRecord(
            ts=time.time(),
            method="auto_compact",
            before_tokens=self._last_usage.messages if hasattr(self, '_last_usage') else 0,
            after_tokens=self._estimate_conv_tokens(),
        ))

        # ⑤ 防漂移：重读最近文件
        await self._anti_drift()
        return True

    async def _anti_drift(self) -> None:
        """防漂移：重读最近操作的 N 个文件（预算 ~50K tokens），重注入 AGENTS.md / Plan / Skills。"""
        # 重读最近文件（从 recent_files 去重取前 5 个）
        read_files = []
        seen = set()
        for f in self.recent_files:
            if f not in seen and len(seen) < 5:
                seen.add(f)
                try:
                    content = Path(f).read_text(encoding="utf-8", errors="replace")
                    read_files.append(f"=== {f} ===\n{content[:10000]}")
                except Exception:
                    pass

        if read_files:
            note = "\n\n[Anti-Drift] 以下为最近操作的文件最新内容，确保你不会丢失当前工作上下文：\n" + "\n".join(read_files)
            self.conv.append(Message(role="user", content=note))
```

### 接口与常量（落到 `agent/context/compactors/auto_compact.py`）

- `class AutoCompact`：
  - `__init__(model, max_failures=3)`：`model` 需有 `async act(messages, tools) -> Decision` 方法（满足 `Model` 协议）。
  - `async compact(conv, boundary) -> list[Message]`：若 boundary ≤0 或 failure_count ≥ max_failures 则原样返回 `conv`；否则提取 `conv[:boundary]`、格式化、调 LLM 生成摘要、解析 `<summary>` 标签、返回 `[summary_msg] + conv[boundary:]`（**新列表**，非原地修改）。
  - `async _call_model(prompt) -> str | None`：组装 system+user 消息 → `model.act()` → 提取 `<summary>...</summary>` 内容 → 无标签时全文兜底 → 异常时返回 None。
  - `_format_history(messages) -> str`：每条消息 `[索引][role] content[:2000]`，tool_calls 追加 `→ tool: name(args)`。
- 常量：`COMPACT_SYSTEM_PROMPT`（内联字符串）、`COMPACT_USER_TEMPLATE`（含 9 段要求 + `{history_text}` 占位符）。

- `track_file_access(path: str) -> None`：追加到 `recent_files`，超 10 个裁掉最早（由 loop._exec_tools 或 transport 在文件读写工具执行后回调）。
- `async compact() -> bool`：完整流程入口：

### 关键决策 / 踩坑（M4.4 务必先看）

1. **失败断路器使用 `>=` 而非 `>`**（偏离草稿）：草稿写 `if self.failure_count > self.max_failures`。当 `failure_count=3` 且 `max_failures=3` 时，条件为 False → 仍会尝试第 4 次调用（失败后 `failure_count` 变为 4），不符合「3 次后放弃」的预期。落地改为 `>=`，使 `failure_count=3` 时直接跳过，不再尝试。
2. **`compact()` 内 `should_compact()` 检测边界后的活跃区**：Microcompact 不压缩非 tool 消息（如 user 指令），`should_compact()` 只计量 `conv[compact_boundary:]`。因此**大消息必须在 boundary 之后**才能触发 auto compact——测试需注意这一点，否则 `should_compact()` 返回 False 导致流程跳过。
3. **`AutoCompact.compact` 返回新列表而非原地修改**：它与 Microcompact 不同（Microcompact 是原地替换 content + 返回同一引用）。`compact()` 内部用 `conv[:boundary]` 切片+ `conv[boundary:]` 切片构造新列表，所以 `ContextManager.compact()` 用 `new_conv is not self.conv` 判断是否发生了压缩（语义正确）。
4. **`_format_history` 截断 2000 字符/条**：避免格式化后的文本超过 LLM 的输入窗口。`tool_calls` 的 arguments 不做截断（通常较小）。
5. **`<summary>` 标签解析**：对齐 Claude Code 输出格式。模型可能输出分析过程后跟 `<summary>...</summary>`，或完全不输出标签（兜底取全文）。
6. **防漂移使用 `errors="replace"`**：防止个别文件编码问题导致整个压缩流程崩溃。
7. **contextvars 隐式 parent 传递（同期优化）**：在 M4.3 开发会话中同步实施了 `agent/obs/tracer.py` 的 contextvars 改造，用 `contextvars.ContextVar("_current_span")` 实现隐式 span parent 继承。**直接受益 M4.3**——`AutoCompact._call_model()` → `model.act()` 调用链无需任何改动即可自动成为 compact span 的子 span（以前需要手动传 `parent` 参数）。详见 `knowledge/INDEX.md` 的「M4.3 同期」小节。
8. **压缩流程已完整 trace 埋点（落地）**：`ContextManager.__init__` 与 `AutoCompact.__init__` 均接受 `tracer: Tracer | None = None`（缺省 None 降级 no-op，不影响无 tracer 的用例）。`ContextManager.compact()` 外包 `context.compact` span；`apply_microcompact()` 内包 `compact.microcompact` span；`AutoCompact.compact()` 内包 `compact.auto_compact` span（记录 boundary / skip / summary_len）；`_anti_drift()` 内包 `compact.anti_drift` span（仅在确有重读文件时）；`AutoCompact._call_model()` 内包 `model.act` span 并把 `decision.usage` 写入 span meta。**关键**：`model.act` 必须挂在 `compact.auto_compact` 之下（而非直接挂 `context.compact`）—— 因为 `AutoCompact.compact()` 的 `compact.auto_compact` span 已在 contextvar 栈中，`_call_model()` 内的 `model.act` 自动继承它作父 span。三者必须注入**同一个 Tracer 实例**才能在同一棵 span 树上汇聚。
