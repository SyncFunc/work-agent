# Step M4.6 CLI 命令（/context + /compact + 状态栏）

## 实现方案

**目标**：在 CLI chat 模式中新增 `/context` 命令查看上下文占用明细、`/compact` 命令手动触发压缩、状态栏实时显示上下文占比。

- `agent/runtime/terminal_transport.py`（TerminalTransport 新增状态栏上下文占比显示）

**关键接口**：

### 6.1 /context 命令

```python
# agent/cli.py chat 分支新增

if cmd in {"/context"}:
    if session.context_mgr is not None:
        usage = session.context_mgr.estimate_usage()
        typer.echo(f"📊 上下文占用明细：", err=True)
        typer.echo(f"  System Prompt 静态段：{usage.system_fixed:,} tokens", err=True)
        typer.echo(f"  System Prompt 动态段：{usage.system_dynamic:,} tokens", err=True)
        typer.echo(f"  Tools 列表：          {usage.tools:,} tokens", err=True)
        typer.echo(f"  对话历史：            {usage.messages:,} tokens", err=True)
        typer.echo(f"  ─────────────────────────", err=True)
        typer.echo(f"  总占用：              {usage.total:,} / {session.context_mgr.effective_window:,} tokens", err=True)
        typer.echo(f"  剩余可用：            {usage.available:,} tokens", err=True)
        typer.echo(f"  使用率：              {usage.used_pct:.1%}", err=True)
        if session.context_mgr.history:
            last = session.context_mgr.history[-1]
            typer.echo(f"  上次压缩：            {last.method} ({last.before_tokens:,} → {last.after_tokens:,})", err=True)
    else:
        typer.echo("→ 上下文管理未启用", err=True)
    continue
```

### 6.2 /compact 命令

```python
# agent/cli.py chat 分支新增

if cmd in {"/compact"}:
    if session.context_mgr is not None:
        typer.echo("→ 正在压缩上下文...", err=True)
        success = await asyncio.to_thread(session._manual_compact)  # 或用 asyncio 包装
        # 实际应为 async：success = await session.context_mgr.compact()
        # 但 chat 循环当前是同步，需要处理
        if success:
            usage = session.context_mgr.estimate_usage()
            typer.echo(f"✅ 压缩完成。当前占用：{usage.total:,} / {usage.effective_window:,} ({usage.used_pct:.1%})", err=True)
        else:
            typer.echo("⚠️ 压缩未执行（可能是上下文尚小或连续失败已放弃）", err=True)
    else:
        typer.echo("→ 上下文管理未启用", err=True)
    continue
```

**关于异步的兼容性**：chat 循环当前在同步代码块内（`while True: task = typer.prompt(...)`）。`/compact` 需要异步调用 `context_mgr.compact()`。有两种方案：
1. **`asyncio.run()` 嵌套**（最简单）：`asyncio.run(session.context_mgr.compact())`——当前 `session.step` 已在 `asyncio.run` 中运行，但 chat 循环每轮结束后事件循环已关闭，可以新开。
2. **持久事件循环**（更优雅）：chat 启动时创建并保持一个事件循环，`step` 和 `/compact` 都在此循环内 `await`。

推荐方案 1（简单，对齐 `step` 已有的 `asyncio.run` 模式）。

### 6.3 状态栏上下文占比

在 `TerminalTransport` 渲染的流式输出中嵌入上下文占比信息（不阻塞主流程）：

```python
# agent/runtime/terminal_transport.py

class TerminalTransport:
    def __init__(self, ..., context_mgr=None):
        ...
        self._context_mgr = context_mgr

    def _status_line(self) -> str:
        """生成状态栏信息（上下文占比 / 模式等）。"""
        parts = []
        if self._context_mgr is not None:
            usage = self._context_mgr.estimate_usage()
            pct = usage.used_pct
            if pct > 0.9:
                color = "red"
            elif pct > 0.7:
                color = "yellow"
            else:
                color = "green"
            parts.append(f"ctx: [{color}]{pct:.0%}[/{color}]")
        return " | ".join(parts)
```

状态栏可以显示在 `typer.prompt` 的提示前缀中，或作为流式输出的附加信息（类似 Claude Code 的状态栏）。本项目采用**在 chat 循环的 prompt 前显示**：

```python
# agent/cli.py chat 循环

while True:
    status = transport._status_line()
    prefix = f"[{status}] you" if status else "you"
    try:
        task = typer.prompt(prefix)
    except (EOFError, KeyboardInterrupt):
        break
```

### 6.4 ContextManager 工具方法

```python
# agent/context/manager.py 新增

class ContextManager:
    def format_usage(self) -> str:
        """返回可读的上下文占用摘要（供 CLI 渲染）。"""
        usage = self.estimate_usage()
        lines = [
            f"📊 上下文占用明细：",
            f"  固定底座：{usage.system_fixed + usage.system_dynamic + usage.tools:,} tokens",
            f"  对话历史：{usage.messages:,} tokens",
            f"  总占用：{usage.total:,} / {self.effective_window:,} ({usage.used_pct:.1%})",
            f"  剩余可用：{usage.available:,} tokens",
        ]
        if self.history:
            last = self.history[-1]
            lines.append(f"  上次压缩：{last.method} ({last.before_tokens:,} → {last.after_tokens:,})")
        return "\n".join(lines)
```

## 知识沉淀

- **`/context` 命令**：chat REPL 中 `cmd == "/context"` → 若 `session.context_mgr` 非空调用 `context_mgr.format_usage()`（err 输出）；否则提示"上下文管理未启用（settings.context.*_enabled 全为 false）"。
- **`/compact` 命令**：`cmd == "/compact"` → `await session.context_mgr.compact()`；成功后**以新 conv 替换会话历史** `session.messages = session.context_mgr.conv`（与 `Session.step` 末尾逻辑一致，保证压缩后主线对话与 ContextManager 同步）。聊天 REPL 本就是 `async def _chat_repl`，故直接 `await`，无需 `asyncio.run()` 嵌套。
- **状态栏**：`TerminalTransport` 新增可选 `context_mgr` 构造参数 + `_status_line()` 方法，返回 `ctx: [color]NN%[/color]` 富文本；占比 >90% 红、>70% 黄、否则绿；无 `context_mgr` 时返回空串。`chat`/`run` 在创建 `TerminalTransport` 时注入 `session.context_mgr`。REPL 每轮 `prompt_async(f"{prefix}: ")` 的 `prefix` 由 `_status_line()` 拼接（`[ctx: …] you`）。
- **`ContextManager.format_usage()`**：返回多行摘要（静态段/动态段/Tools/对话历史/总占用/剩余/使用率 + 最近一次 `history` 压缩记录），供 CLI 与测试复用。
- **改动文件**：`agent/cli.py`（/context /compact 命令 + 状态栏前缀 + transport 注入 context_mgr + docstring）、`agent/runtime/terminal_transport.py`（`context_mgr` 参数 + `_status_line()`）、`agent/context/manager.py`（`format_usage()`）、`tests/test_cli.py`（5 个 M4.6 用例）。
- **注意**：`ContextUsage` 只有 `total`/`used_pct` 等字段，`effective_window` 在 `ContextManager` 上；CLI 打印当前占用时应取 `session.context_mgr.effective_window` 而非 `usage.effective_window`。

### 事后修复（post-M4.7，2026-07-21）

- **状态栏原始标签渲染**：`prompt_async(f"{prefix}: ")` 中 prompt_toolkit 不渲染 rich 标记，原样打印 `[green]10%[/green]`。修复为交互模式用 `prompt_toolkit.formatted_text.FormattedText` 着色（`green→ansigreen` 等），非交互模式用正则剥离标记显示纯文本 `ctx: 10% | you`。
- **`/context` 动态段恒为 0**：`_system_dynamic` 从未被写入。`agent/core/prompts.py` 抽出 `_build_system_parts()` 返回 `(static, dynamic)`；`AgentLoop._system_prompt()` 每次构建 system prompt 时把 `_estimate_tokens(static/dynamic)` 回填 `context_mgr._system_fixed/_system_dynamic`。
- **exit 时后台 Subagent 残留**：`Session.shutdown_background(timeout=30)` 优雅关闭（等待完成，超时则 cancel 并 gather 收尾）；`agent/cli.py` chat 退出前提示并调用，避免被 `asyncio.run` 粗暴取消导致工具中断。
