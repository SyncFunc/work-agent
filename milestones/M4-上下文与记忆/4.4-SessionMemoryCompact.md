# Step M4.4 Session Memory Compact（零成本首选）

## 实现方案

**目标**：实现 Session Memory Compact 压缩器——后台**增量维护**一个会话摘要文件（markdown，10 段固定结构），每次压缩时**优先复用**这份摘要替代历史，零 API 调用。这是 Claude Code 的"零成本首选"方案，比 AI 摘要压缩（3b）更划算。

> 调研依据：`knowledge/claude-code-context-management.md` §3.3a / §3.5，以及《Claude Code Session Memory 深度解析》（claudefa.st/blog/guide/mechanics/session-memory）。

**改动文件**：
- `agent/context/compactors/session_memory.py`（新建，`SessionMemory` 类）
- `agent/context/manager.py`（`ContextManager` 集成 Session Memory 优先检查 + 瞬时 compact）
- `agent/config/settings.py`（新增 Session Memory 阈值配置）
- `tests/test_context.py`（追加测试用例）

---

### 4.4.1 摘要文件结构（10 段固定 section）

Session Memory 的摘要不是自由文本，而是 **10 个固定 section** 的 markdown；每个 section 最大 **2,000 tokens**，整个文件最大 **12,000 tokens**（超限时 AI 被强制压缩旧内容）：

```markdown
# Session Title       - 5-10 词信息密集描述
# Current State       - 正在进行的工作、未完成任务、下一步
# Task specification  - 用户要求构建什么、设计决策
# Files and Functions - 重要文件及其作用
# Workflow            - 常用命令及执行顺序
# Errors & Corrections- 遇到的错误及修复方法
# Codebase and System Documentation - 系统组件
# Learnings          - 有效/无效方法
# Key results        - 用户要求的具体输出（完整表格、答案等）
# Worklog            - 逐步行动记录
```

本项目把这份摘要存为 `<project>/.agent/sessions/<session_id>/session-memory/summary.md`（每个 session 独立目录，目录权限 0o700、文件 0o600，摘要含项目敏感信息）。

---

### 4.4.2 触发机制（增量更新阈值）

Session Memory 的提取由**后处理钩子**驱动（等价于本项目 `Session.step()` 结束后调用 `update_session_memory()`）。核心判断 `should_update()`，**必要条件：token 增量阈值始终要求满足**：

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `minimum_message_tokens_to_init` | **10,000** | 初次触发所需的上下文 token 数阈值（短会话 < 10K 不提取） |
| `minimum_tokens_between_update` | **5,000** | 两次更新之间最小 token 增量 |
| `tool_calls_between_updates` | **3** | 两次更新之间最少 tool call 次数 |

触发条件（两者满足其一）：
1. token 增量 ≥ `minimum_tokens_between_update` **AND** tool call 次数 ≥ `tool_calls_between_updates`；
2. token 增量 ≥ `minimum_tokens_between_update` **AND** 最后一轮没有 tool call（自然对话断点）。

> **前置条件**：Session Memory 要求 **Auto Compact（M4.3）必须开启**，否则直接跳过（`if not auto_compact_enabled: return`）。语义上：没有自动压缩需求，就不需要后台记忆。

**提取过程（不阻塞主线程）**：摘要更新通过一个**独立的"记忆子 agent"**运行，该子 agent **只被授权用 Edit 工具修改 summary.md**，其他工具一律拒绝——保证记忆写入不会越权去改项目代码。本项目可复用 M5 的 Subagent 隔离机制实现，且**串行化**（同时只有一个提取在运行）。

### 4.4.3 关键接口

```python
# agent/context/compactors/session_memory.py

# 10 段固定 section 模板（供提取 prompt 与解析）
SUMMARY_SECTIONS = [
    "Session Title", "Current State", "Task specification",
    "Files and Functions", "Workflow", "Errors & Corrections",
    "Codebase and System Documentation", "Learnings",
    "Key results", "Worklog",
]
SECTION_MAX_TOKENS = 2_000
SUMMARY_MAX_TOKENS = 12_000

@dataclass
class SessionMemoryData:
    """会话摘要文件内容。"""
    session_id: str
    created_at: float
    updated_at: float
    summary: str  # 结构化摘要文本（10 段 markdown 格式）
    version: int = 1
    stats: dict = field(default_factory=dict)  # 压缩次数/节省 token/提取次数

@dataclass
class SessionMemoryConfig:
    session_memory_dir: str = ".agent/sessions"
    minimum_message_tokens_to_init: int = 10_000
    minimum_tokens_between_update: int = 5_000
    tool_calls_between_updates: int = 3
    enabled: bool = True  # 前置：要求 Auto Compact 开启

class SessionMemory:
    """Session Memory Compact 压缩器：零成本，复用后台增量维护的摘要文件。"""

    def __init__(self, config: SessionMemoryConfig, session_id: str = ""):
        self.config = config
        self.session_id = session_id
        self._base = Path(config.session_memory_dir) / session_id / "session-memory"
        self._base.mkdir(parents=True, exist_ok=True)
        os.chmod(self._base, 0o700)  # 目录仅本人可读

    @property
    def _summary_path(self) -> Path:
        return self._base / "summary.md"

    def load(self) -> str | None:
        """加载会话摘要。不存在或为空返回 None。"""
        if not self._summary_path.exists():
            return None
        text = self._summary_path.read_text(encoding="utf-8").strip()
        return text or None

    def save(self, summary: str, stats: dict | None = None) -> None:
        """保存/更新会话摘要（10 段 markdown）。"""
        data = SessionMemoryData(
            session_id=self.session_id,
            created_at=time.time(),
            updated_at=time.time(),
            summary=summary,
            stats=stats or {},
        )
        self._summary_path.write_text(
            json.dumps(asdict(data), ensure_ascii=False, indent=2),
            encoding="utf-8",
        )
        os.chmod(self._summary_path, 0o600)  # 文件仅本人可读

    def should_update(self, conv_tokens: int, tokens_since_update: int,
                      tool_calls_since_update: int, last_round_has_tool: bool) -> bool:
        """后处理钩子判断：是否该增量更新摘要。token 增量为必要条件。"""
        if not self.config.enabled:
            return False
        if self.load() is None and conv_tokens < self.config.minimum_message_tokens_to_init:
            return False  # 初次需达到 init 阈值
        if tokens_since_update < self.config.minimum_tokens_between_update:
            return False
        if tool_calls_since_update >= self.config.tool_calls_between_updates:
            return True
        return not last_round_has_tool  # 自然对话断点

    def compact(self, conv: list[Message], boundary: int,
                keep_recent_tokens: int = 10_000,
                min_recent_messages: int = 5,
                max_recent_tokens: int = 40_000) -> list[Message] | None:
        """Session Memory Compact：直接用摘要替代 boundary 前历史（瞬时，零 API 调用）。

        保留最近原文的约束（对齐 Claude Code DEFAULT_SM_COMPACT_CONFIG）：
        - 至少保留 keep_recent_tokens（默认 10K）tokens 原文
        - 至少保留 min_recent_messages（默认 5）条含文本消息
        - 至多保留 max_recent_tokens（默认 40K） tokens
        有摘要则返回压缩后 conv，否则返回 None（走 Auto Compact 兜底）。
        """
        summary = self.load()
        if summary is None:
            return None

        # 从 boundary 之后向前保留最近原文，满足最小保留约束
        recent, recent_tokens, recent_msgs = [], 0, 0
        for msg in reversed(conv[boundary:]):
            text = msg.content or ""
            t = _estimate_tokens(text)
            reached_min = recent_msgs >= min_recent_messages and recent_tokens >= keep_recent_tokens
            if recent_tokens + t > max_recent_tokens or (reached_min and recent_tokens + t > keep_recent_tokens):
                break
            recent.insert(0, msg)
            recent_tokens += t
            if text.strip():
                recent_msgs += 1

        return [Message(role="user", content=f"[Session Summary]\n{summary}")] + recent
```

### 4.4.4 集成到 ContextManager（压缩决策流增强）

```python
async def compact(self) -> bool:
    # ① Microcompact（零成本）
    self.apply_microcompact()
    if not self.should_compact():
        return True
    # ③a 优先 Session Memory Compact（瞬时，零 API 调用）
    if self.session_memory is not None:
        sm_result = self.session_memory.compact(self.conv, self.compact_boundary)
        if sm_result is not None:
            self.conv = sm_result
            self.mark_boundary()
            await self._anti_drift()
            return True
    # ③b Auto Compact（1 次调用，兜底）
    ...
```

**瞬时 `/compact`**：当 Session Memory 已启用且摘要就绪，`/compact` 命令**直接加载 summary.md 作为压缩结果**，无需重新分析整个对话（传统 AI 摘要约需 2 分钟）。实现即调用上面 `compact()`，若返回 `None` 再降级到 Auto Compact。

---

### 4.4.5 增量更新触发时机（提取过程）

- 每次 `Session.step()` 结束后，计算本轮 `tokens_since_update` / `tool_calls_since_update` / `last_round_has_tool`，调用 `session_memory.should_update(...)`。
- 命中则**启动一个独立的"记忆子 agent"**（只授权 Edit summary.md，不授权读写项目代码），基于当前 `conv` 增量更新 `summary.md`；该过程异步、串行化、不阻塞主对话。
- 首次 Auto Compact 后也可把 9 段摘要**反填**进 Session Memory，作为初始摘要。

---

### 4.4.6 跨会话 Recall（M5/M6 协同）

新会话启动时，系统查找该项目目录下**历史 session 的 summary.md**，将内容注入上下文，并附标注：*"来自过去会话的背景参考，不一定与当前任务相关"*，避免模型盲目遵循旧决策。终端可提示 `Recalled N memories`。

---

### 4.4.7 与 `/remember` → AGENTS.local.md 提升通道

Session Memory 是"自动、临时的历史快照"，AGENTS.md 是"手动确认、持久的项目规则"。`/remember` 命令识别跨 session 的重复模式，提议把内容**提升**写入 `AGENTS.local.md`（或项目级 AGENTS.md）永久保留。

| 特性 | Session Memory | AGENTS.md |
|------|---------------|-----------|
| 创建者 | Agent（自动） | 用户（手动） |
| 范围 | 每个 session 快照 | 持久项目规则 |
| 优先级 | 背景参考 | 高优先指令 |
| 最佳用途 | 跨 session 连续性 | 标准规范、架构、命令 |

---

**关键决策**：
- **零成本首选**：Session Memory（3a）优先于 Auto Compact（3b）；有摘要直接复用，不花 API 调用。
- **固定 10 段结构**：摘要按 10 段 markdown 维护，便于解析、增量更新与跨会话召回。
- **保留原文细节**：保留最近原文（默认 10K~40K tokens，至少 5 条消息），避免丢失当前工作细节。
- **前置依赖 Auto Compact**：未开启自动压缩则不启用 Session Memory。
- **提取隔离**：记忆子 agent 仅能 Edit summary.md，不碰项目代码（安全）。

**依赖/环境**：`ContextManager`（M4.1）+ `Microcompact`（M4.2）+ `AutoCompact`（M4.3）；`session_id`；M5 Subagent 隔离（用于记忆子 agent，可选）。

**注意事项**：
- 短会话（< 10,000 tokens）不会触发任何提取。
- 提取应串行化（同时只有一个提取在运行），避免并发写 summary.md。
- 子 agent / teammate 等非主线程查询**不触发** Session Memory 提取。

## 验收标准

- [ ] `SessionMemory` 未保存时 `load()` 返回 `None`。
- [ ] `save("summary")` 后 `load()` 返回 `"summary"`；写出的 `summary.md` 文件权限为 0o600、目录 0o700。
- [ ] `should_update()`：未开启 `enabled` 返回 `False`；初次会话 token < 10K 返回 `False`；token 增量 < 5K 返回 `False`；token 增量达标且 tool call ≥ 3 返回 `True`；token 增量达标且最后一轮无 tool call 返回 `True`。
- [ ] `compact(conv, boundary)` 有摘要时返回 `[summary_msg] + recent_msgs`。
- [ ] `compact()` 无摘要时返回 `None`（触发 Auto Compact 兜底）。
- [ ] `compact()` 保留的原文满足：≥ `keep_recent_tokens` 或 ≥ `min_recent_messages` 条消息，且 ≤ `max_recent_tokens`（估算值 ±10%）。
- [ ] `ContextManager.compact()` 优先走 Session Memory 分支（有摘要时）。
- [ ] `SessionMemory` 与 `ContextManager` 集成正确（瞬时 `/compact` 路径）。
- [ ] 测试：`tests/test_context.py` 追加 ≥10 用例覆盖 load/save/权限/should_update 四情形/compact 有摘要/compact 无摘要/保留原文约束/增量更新。
