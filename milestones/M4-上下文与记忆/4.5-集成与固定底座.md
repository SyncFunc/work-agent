### 5.1 AGENTS.md 项目级记忆文件

**行为约定**（对齐 Claude Code 的 CLAUDE.md）：
- 文件**永不进摘要**：每次投影时从磁盘重新读取，注入到 messages 首条 `<system-reminder>`。

- 位置优先级：项目根 `AGENTS.md` > `~/.agent/AGENTS.md`（用户级）。
- `Settings.context.agents_md_enabled` 控制开关。

### 5.2 System Prompt 重构（静态/动态分离）

```python
# agent/core/prompts.py 更新

def build_system_prompt(settings, *, plan_mode, has_plan, clarify_enabled, context):
    """构建完整 system prompt（静态段 + 动态段）。

    静态段（可 prompt caching，scope='global'）：
    - 身份/安全约束
    - 工具规范
    - 语气/输出效率

    动态段（每轮更新，不缓存）：
    - 当前日期
    - Git 仓库状态
    - 会话指引（当前模式/计划状态）
    - Memory（AGENTS.md 内容）
    - MCP 指令
    """
    static = load_prompt("system_static").render(...)
    dynamic = _build_dynamic_segment(settings, plan_mode=plan_mode, has_plan=has_plan)
    return static + "\n" + dynamic

def _build_dynamic_segment(settings, *, plan_mode, has_plan):
    """构建动态段，含日期、Git 状态、AGENTS.md。"""
    parts = []

    # 日期（移出静态段，改善缓存命中率）
    from datetime import date
    parts.append(f"## 当前日期\n{date.today().isoformat()}\n")

    # AGENTS.md
    if settings.context.agents_md_enabled:
        agents_content = _read_agents_md()
        if agents_content:
            parts.append(f"## AGENTS.md\n{agents_content}\n")

    # Git 仓库状态
    try:
        import subprocess
        branch = subprocess.check_output(["git", "branch", "--show-current"], text=True).strip()
        status = subprocess.check_output(["git", "status", "--short"], text=True).strip()
        parts.append(f"## Git 状态\n分支：{branch}\n{status[:1000]}\n")
    except Exception:
        pass

    return "\n".join(parts)
```

### 5.3 ContextManager 集成到 Session 与 Loop

```python
# agent/core/session.py 更新

class Session:
    def __init__(self, model, reg, settings, tracer=None, *, plan_mode=False, plan_path=None, trace_store=None, context_mgr=None):
        ...
        self.context_mgr = context_mgr  # 可选注入

    async def step(self, task, transport, *, yes=False, fatal_plan_decline=False):
        current_task = task
        while True:
            # 每轮 step 前：设置 ContextManager 的 conv 引用
            if self.context_mgr is not None:
                self.context_mgr.set_conv(self.messages)

            res = await self.loop.run(current_task, self.messages, ...)
            self.messages = list(res.messages or self.messages)

            # 每轮 step 后：检查是否需要压缩
            if self.context_mgr is not None:
                self.context_mgr.set_conv(self.messages)
                if self.context_mgr.should_compact():
                    await self.context_mgr.compact()
                    self.messages = self.context_mgr.conv  # 替换为压缩后的 conv

            ...


# agent/core/loop.py 更新

class AgentLoop:
    async def run(self, task, messages=None, *, ..., context_mgr=None):
        ...
        conv = list(messages) if messages else []
        conv.append(Message(role="user", content=task))

        # 每轮 _decide 前：执行 Microcompact（零成本）
        if context_mgr is not None:
            context_mgr.set_conv(conv)
            context_mgr.apply_microcompact()
            conv = context_mgr.conv

        ...

    async def _exec_tools(self, tool_calls, stream, context_mgr=None):
        ...
```

### 5.4 prompt caching 支持

在构建 system prompt 时标记静态段可缓存：

```python
# agent/core/prompts.py

# 静态段的内容（身份/安全/工具规范）作为稳定前缀，
# 发送给模型时放在 messages 首条，利用 API 的 prompt caching 机制。
# DeepSeek / OpenAI 自动对前缀做 caching，我们只需保证前缀稳定：
# 1. 日期不放在静态段（已外移到动态段）
# 2. AGENTS.md 内容也放在动态段（因可能被用户编辑）
# 3. 静态段只放"不会变"的部分
```

## 知识沉淀

- **`AGENTS.md` 命名约定**：对齐 Claude Code 的 `CLAUDE.md` → 本项目用 `AGENTS.md`（项目根，用户可编辑，应 commit）。
- **AGENTS.md 位置优先级**：项目根 `<AGENT_PROJECT_ROOT>/AGENTS.md` > 项目级 `<root>/.agent/AGENTS.md` > 用户级 `<AGENT_USER_CONFIG_DIR or ~/.agent>/AGENTS.md`。（`agents_md_enabled=False` 时不读不注入。）
- **AGENTS.md 永不压缩**：每次投影时从磁盘重新读取，注入到 system prompt 动态段首条 `<system-reminder>`；不属于被压缩的 conv 历史。
- **System Prompt 静态/动态分离**：静态段 = `system.md` 渲染（身份/安全/工具规范/模式指引/skill·agent 目录），稳定前缀在前以复用 prompt cache；动态段 = 日期 + Git 状态 + AGENTS.md，每轮重新生成追加在后。验证：两日期下静态前缀完全一致。
- **日期外移出静态段** → 避免每轮变化破坏稳定前缀的 prompt caching 命中。
- **`build_context_manager(settings, model, *, session_memory, tracer)` 工厂**：唯一集成入口。按 `context` 子开关决定启用：
  - `microcompact_enabled` → 构造 `Microcompact`；否则传 `None`（显式禁用）。
  - `auto_compact_enabled` → 构造 `AutoCompact(model)`；否则 `None`。
  - `session_memory` 由 `Session` 依据 `session_memory_enabled`（且依赖 `auto_compact_enabled`）先构造后传入。
- **⚠️ 坑：`_UNSET` 哨兵区分"缺省"与"显式禁用"**：`ContextManager.__init__` 的 `microcompact` 默认用模块级哨兵 `_UNSET`，缺省→构造默认 `Microcompact`，显式 `None`→禁用。若用 `None` 当默认值，会把"禁用"误判为"未指定"而重新构造，导致 `build_context_manager(microcompact_enabled=False)` 仍得到启用实例。
- **⚠️ 坑：压缩 span 必须挂 `Session.root_span` 下**：`ContextManager` 新增 `root_span` 字段，`apply_microcompact` 与 `compact` 的 `_span` 调用传 `parent=self.root_span`。否则 microcompact span 会成为新的 root span，破坏"每个 Session 仅一个 root span"的可观测性契约（回归 `test_session_shares_single_root_span`）。`Session` 构建 `context_mgr` 后执行 `self.context_mgr.root_span = self.root_span`。
- **ContextManager 集成点**：`Session.step()` 每轮前 `set_conv(messages)`、每轮后 `should_compact()`→`compact()`→`messages = context_mgr.conv`；`Loop.run()` 每轮 `_decide` 前对 conv 投影 `apply_microcompact()`（绝不碰 EventStream 审计真相）；`Loop._exec_tools` 对 `read/write/edit` 调用 `track_file_access(path)` 供防漂移。
- **全关零开销**：`microcompact_enabled`、`auto_compact_enabled`、`session_memory_enabled` 全为 `False` 时 `Session` 不构建 `context_mgr`（保持 `None`），主流程 seam 全部短路。
