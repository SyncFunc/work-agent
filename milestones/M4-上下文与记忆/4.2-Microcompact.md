# Step M4.2 Microcompact（零成本）

## 实现方案

**目标**：实现 Microcompact 压缩器——在每次 API 请求前，把 `conv` 中**边界之前**较旧的 `role="tool"` 消息的 `content` 替换为占位符 `[Old tool result content cleared]`，保留最近 N 个结果不动。零 API 调用，消息条数不变，不拆散 `tool_use`/`tool_result` 配对。

**改动文件**：
- `agent/context/compactors/microcompact.py`（新建）
- `agent/context/manager.py`（ContextManager 集成 Microcompact 调用入口）
- `tests/test_context.py`（追加测试用例）

**关键接口**：

```python
# agent/context/compactors/microcompact.py

COMPACTABLE_TOOLS = {"bash", "read", "grep", "glob", "write", "edit", "find"}
"""可压缩的工具结果类型（大输出类工具）。"""

PLACEHOLDER = "[Old tool result content cleared]"
"""占位符字符串，对齐 Claude Code 行为。"""

class Microcompact:
    """Microcompact 压缩器：零成本，仅替换旧 tool_result 内容为占位符。

    策略：
    - 只操作 compact_boundary 之前的消息（已被压缩过的不再处理）。
    - 只操作 conv 中 role="tool" 的消息。
    - 保留最近 keep_recent 个 tool_result 不动。
    - 不修改 tool_use 消息（配对完整性靠 tool_call_id 保持）。
    - 消息条数不变，索引不变。
    """

    def __init__(self, keep_recent: int = 5, compactable_tools: list[str] | None = None):
        self.keep_recent = keep_recent
        self.compactable_tools = compactable_tools or list(COMPACTABLE_TOOLS)

    def compact(self, conv: list[Message], boundary: int) -> list[Message]:
        """对 conv 执行 Microcompact，返回新列表（原地修改或新列表）。"""
        if boundary <= 0:
            return conv

        # 统计 boundary 前所有 tool 消息中属于可压缩工具的索引
        tool_indices: list[int] = []
        for i, msg in enumerate(conv[:boundary]):
            if msg.role == "tool" and self._is_compactable(msg):
                tool_indices.append(i)

        # 保留最近 keep_recent 个
        replace_indices = set(tool_indices[:-self.keep_recent]) if len(tool_indices) > self.keep_recent else set()

        for i in replace_indices:
            conv[i] = Message(
                role="tool",
                content=PLACEHOLDER,
                tool_call_id=conv[i].tool_call_id,
            )
        return conv

    def _is_compactable(self, msg: Message) -> bool:
        """判断此 tool 消息的内容是否来自可压缩工具（通过 tool_call_id 交叉验证不可行，
        此处简化：只要有 content 且不太短的 tool 消息均视为可压缩。
        更精确的做法需遍历 assistant 消息交叉 tool_call_id→name，可选增强。"""
        return bool(msg.content) and len(msg.content) > 100
```

**集成到 ContextManager**：

```python
# agent/context/manager.py 新增
from agent.context.compactors.microcompact import Microcompact

class ContextManager:
    def __init__(self, ...):
        ...
        self.microcompact = Microcompact(
            keep_recent=context_config.microcompact_keep_recent,
        )

    def apply_microcompact(self) -> list[Message]:
        """执行 Microcompact，返回处理后的 conv。"""
        if not self.conv:
            return self.conv
        return self.microcompact.compact(self.conv, self.compact_boundary)
```

**关键决策**：
- **只改 content，不动配对**：`tool_call_id` 保持不变，API 协议满足。
- **保留最近 N 个**：默认 5（`keep_recent=5`），Claude Code 也是类似思路——最近的操作结果不应被压缩，否则模型看不到当前正在改的代码。
- **边界限定**：只处理 `compact_boundary` 之前的消息——已被压缩过的历史不重复占位。
- **零成本**：纯字符串替换，不调 LLM。

**依赖/环境**：
- `Message` 数据类，`role="tool"` 和 `tool_call_id` 字段。
- `ContextManager.compact_boundary`（M4.1 实现）。

## 验收标准

- [ ] `Microcompact.compact(conv, 0)` 不修改任何消息。
- [ ] `Microcompact.compact(conv, len(conv))` 把 boundary 前旧 tool 消息替换为占位符，保留最近 5 个。
- [ ] `keep_recent=0` 时替换所有旧 tool 消息。
- [ ] 占位后 `tool_call_id` 不变，`role` 仍为 `"tool"`。
- [ ] `Message` 内容长度 > 100 才被替换，短消息不动。
- [ ] 非 `role="tool"` 的消息不被修改。
- [ ] `ContextManager.apply_microcompact()` 返回正确结果。
- [ ] 测试：`tests/test_context.py` 追加 ≥8 用例覆盖边界 / keep_recent / 占位符 / 非 tool 不动。
