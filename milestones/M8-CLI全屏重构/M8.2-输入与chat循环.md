# Step M8.2 输入与 chat 循环

## 实现方案

- 目标：接入用户输入（`TextArea`）+ 提交 → 用 **thread worker** 跑 `session.step`，把旧 `_chat_repl`
  的 `asyncio.run` 挪进独立线程 loop（调研 §4.2 方案 B）；`/命令` 复用 `dispatch_command`。
- 改动文件：
  - `agent/tui/app.py`：
    - 底部 `TextArea`（多行）+ 提交绑定（`Ctrl+J` 提交；`Enter` 在非空时提交；`Tab` 补全）。
    - `on_mount` 起 thread worker：`self.run_worker(self._session_driver, thread=True)`；
      worker 内 `new_loop = asyncio.new_event_loop(); asyncio.set_event_loop(new_loop); new_loop.run_forever()`，
      并起任务消费 `self._task_queue`。
    - `_session_driver`：`while True: task = await self._task_queue.get(); await session.step(task, transport=self.transport, ...)`
      （逻辑几乎等同旧 `_chat_repl`，仅运行位置从主线程 asyncio 改为 worker 线程 loop）。
    - 提交回调：把 `TextArea` 内容 `put` 进 `self._task_queue`（经 `call_from_thread` 从主线程投递到 worker loop）。
    - `/` 开头走 `dispatch_command`（现有命令分发，复用，不改）。
  - `agent/cli.py`：`chat --tui` 把构造好的 `Session` 注入 `ChatApp` 后 `app.run()`。
- 关键接口/算法：
  - `ChatApp.run_worker(coro, thread=True)` 返回 worker；worker 内独立 loop（方案 B）。
  - `self._task_queue = asyncio.Queue()` 在 worker loop 创建。
- 依赖/环境：M8.1 的 `TextualTransport` 已就绪；`Session` 由 `cli` 构造（现有 Session 工厂）。

## 验收标准

- [ ] `test_input_drives_step`：`run_test` 下向 `TextArea` 写入 "hello" 并提交，断言 worker 被驱动、
  `session.step` 经 `FakeModel` 返回，主区出现助理回复文本。
- [ ] `test_slash_command`：输入 `/context` 等命令，断言触发 `dispatch_command` 对应分支（spy 验证）。
- [ ] 不变量：Textual 自带 loop 未被内层 `asyncio.run` 破坏（方案 B）；旧 `TerminalTransport._chat_repl`
  不动；`session.step` 签名未变。

## 知识沉淀

- **thread worker 起独立 loop 的最小范式**（`ChatApp._run_worker`）：
  `threading.Thread(target=..., daemon=True)` 内 `loop = asyncio.new_event_loop();
  asyncio.set_event_loop(loop); loop.run_until_complete(self._session_loop())`。
  关键是 **Textual 自带 loop 内绝不 `asyncio.run(session.step)`**——把 step 挪进独立线程 loop。
- **`_task_queue` 跨线程投递**：主线程 `action_submit` 用 `self._task_queue.put(text)`（线程安全
  `queue.Queue`）；worker 用 `await loop.run_in_executor(None, self._task_queue.get)` 取任务，
  再 `await self.session.step(task, self.transport, ...)`。哨兵 `_SENTINEL` 通知退出。
- **`TextArea` 提交 / 键绑定**：绑定 `("ctrl+j", "submit", "发送")` → `async def action_submit`
  （空输入忽略；`exit/quit` → `self.exit()`；`/` 开头 → `await self._handle_command` 走
  `dispatch_command`；否则投递 worker）。`Enter` 保留换行（与 Claude Code 的 Shift+Enter 换行一致）。
- **`Session` 注入 `App`**：`chat --tui` 构造 `Session` 后传入 `ChatApp(session=..., settings=...)`；
  `on_mount` 在有 session 时创建 `TextualTransport(self, context_mgr=session.context_mgr)` 并启动 worker。
- **事件桥接闭环**：worker 内 `session.step` → loop `transport.bind(stream)` 订阅 → 事件在 worker
  线程触发 `on_event` → `TextualTransport._bridge` 因跨线程走 `app.call_from_thread` 把 UI 更新
  投递回主线程。**已验证**：`run_test` + `pilot.press("ctrl+j")` 驱动 FakeModel，主区出现 AssistantMessage。
- **不变量**：`AgentTransport`/`EventStream`/loop 未改；旧 `_chat_repl` 不动；`session.step` 签名不变。
- **踩坑**：worker 线程的 asyncio loop 与 Textual loop 是**两个不同 loop**；HITL（M8.3）的
  `Future` 唤醒必须用线程安全方式（见 M8.3）。
