# Step M8.2 输入与 chat 循环

## 实现方案

- 目标：接入用户输入（`TextArea`）+ 提交 → 用 **thread worker** 跑 `session.step`，把旧 `_chat_repl`
  的 `asyncio.run` 挪进独立线程 loop（调研 §4.2 方案 B）；`/命令` 复用 `dispatch_command`。
- 改动文件：
  - `agent/tui/app.py`：
    - 底部 `TextArea`（多行）+ 提交绑定（`Ctrl+J` 提交；`Enter` 在非空时提交；`Tab` 补全）。
    - `on_mount` 起 thread worker：`self.run_worker(self._session_driver, thread=True)`；
      worker 内 `new_loop = asyncio.new_event_loop(); asyncio.set_event_loop(new_loop); new_loop.run_forever()`，
      并起任务消费 `self._task_queue`。
    - `_session_driver`：`while True: task = await self._task_queue.get(); await session.step(task, transport=self.transport, ...)`
      （逻辑几乎等同旧 `_chat_repl`，仅运行位置从主线程 asyncio 改为 worker 线程 loop）。
    - 提交回调：把 `TextArea` 内容 `put` 进 `self._task_queue`（经 `call_from_thread` 从主线程投递到 worker loop）。
    - `/` 开头走 `dispatch_command`（现有命令分发，复用，不改）。
  - `agent/cli.py`：`chat --tui` 把构造好的 `Session` 注入 `ChatApp` 后 `app.run()`。
- 关键接口/算法：
  - `ChatApp.run_worker(coro, thread=True)` 返回 worker；worker 内独立 loop（方案 B）。
  - `self._task_queue = asyncio.Queue()` 在 worker loop 创建。
- 依赖/环境：M8.1 的 `TextualTransport` 已就绪；`Session` 由 `cli` 构造（现有 Session 工厂）。
