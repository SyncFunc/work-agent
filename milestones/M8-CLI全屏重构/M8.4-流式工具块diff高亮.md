# Step M8.4 流式 + 工具块 + diff 高亮

## 实现方案

- 目标：流式 Markdown（节流 + 仅当前块重渲染）、可折叠工具块（`Collapsible` + `Syntax(json)`）、
  diff 高亮（`Syntax(diff)` 复用 `res.diff`）。对标 Claude Code 工具调用与写文件体验（调研 §2）。
- 改动文件：
  - 新建 `agent/tui/widgets.py`：
    - `AssistantMessage(Static)`：包 `Markdown`；`update_stream(text)` 节流（每 ~100ms 或累积 N 字符）
      `app.call_later(self.update)`，定稿一次性完整 `update`（规避 O(n²) 重解析，调研 §4.4）。
    - `ToolBlock(Collapsible)`：标题 `🔧 name`；展开 `Syntax(ev.params, "json")` + 结果
      `Syntax(res.diff, "diff")` 或 `Markdown`（复用现有 `res.diff`）。
  - `agent/tui/app.py`：`_append_assistant` 创建 `AssistantMessage` 并持有"当前流式块"引用，
    后续 `TEXT` 增量走 `update_stream`；`TOOL_USE` 建 `ToolBlock`，`TOOL_RESULT` 更新之。
- 关键接口/算法：
  - 流式事件假设为多次 `TEXT` 增量 或 单 `ASSISTANT_MESSAGE` 终态 —— 需与 `events.py` 实际流式事件对齐。
  - 节流：用 `app.set_interval` 或自带 `throttle`，避免每次增量整段重解析。
- 依赖/环境：落地时先读 `agent/core/events.py` 确认流式事件类型；`res.diff` 复用现有。

## 验收标准

- [ ] `test_streaming_throttle`：连续 50 次 `TEXT` 增量，断言 `Markdown.update` 调用次数远小于 50
  （节流生效），且最终文本完整。
- [ ] `test_tool_block`：`TOOL_USE` + `TOOL_RESULT` → `RichLog` 出现 `Collapsible` 且含 JSON 高亮与 diff 高亮。
- [ ] 不变量：流式不重刷整屏（仅当前块）；不引入 ptk `Live`（只用 Textual 部件，规避旧踩坑）。
