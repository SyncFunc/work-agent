# Step M8.4 流式 + 工具块 + diff 高亮

## 实现方案

- 目标：流式 Markdown（节流 + 仅当前块重渲染）、可折叠工具块（`Collapsible` + `Syntax(json)`）、
  diff 高亮（`Syntax(diff)` 复用 `res.diff`）。对标 Claude Code 工具调用与写文件体验（调研 §2）。
- 改动文件：
  - 新建 `agent/tui/widgets.py`：
    - `AssistantMessage(Static)`：包 `Markdown`；`update_stream(text)` 节流（每 ~100ms 或累积 N 字符）
      `app.call_later(self.update)`，定稿一次性完整 `update`（规避 O(n²) 重解析，调研 §4.4）。
    - `ToolBlock(Collapsible)`：标题 `🔧 name`；展开 `Syntax(ev.params, "json")` + 结果
      `Syntax(res.diff, "diff")` 或 `Markdown`（复用现有 `res.diff`）。
  - `agent/tui/app.py`：`_append_assistant` 创建 `AssistantMessage` 并持有"当前流式块"引用，
    后续 `TEXT` 增量走 `update_stream`；`TOOL_USE` 建 `ToolBlock`，`TOOL_RESULT` 更新之。
- 关键接口/算法：
  - 流式事件假设为多次 `TEXT` 增量 或 单 `ASSISTANT_MESSAGE` 终态 —— 需与 `events.py` 实际流式事件对齐。
  - 节流：用 `app.set_interval` 或自带 `throttle`，避免每次增量整段重解析。
- 依赖/环境：落地时先读 `agent/core/events.py` 确认流式事件类型；`res.diff` 复用现有。

## 知识沉淀

### ① 流式事件类型（与 `events.py` 对齐）

- 增量：`EventType.TEXT`（`kind="content"` 正式 / `"reasoning"` 思考），逐片推送，`append` 累加。
- 终态：`EventType.DECISION`（一轮决策收尾）→ transport 调 `app.finalize_stream()` 定稿当前块。
- 工具：`TOOL_USE` / `TOOL_RESULT`（`ev.tool_result.diff` 携带 diff），与流式文本解耦。

### ② 节流实现（coalesce，非定时器）

- 基类 `_StreamingMessage`：`append` 只累加 `self._full`，并 `self.app.call_later(self._flush)` 排程一次；
  已排程（`_dirty`）则跳过，把后续增量合并到下一个 loop tick 再整段 `update`（规避 O(n²) 重解析）。
- `app.finalize_stream()` 调 `self._current.flush()` 把剩余未刷增量一次性渲染，保证定稿完整。
- 未挂载 / 无 app 时 `_schedule_flush` 的 `call_later` 异常兜底为同步 `_flush`，文本不丢。

### ③ `ToolBlock` 的 `Syntax` 高亮选择

- 参数：`Syntax(args_json, "json", theme="ansi_dark", word_wrap=True)`（`args` 已由 `app.append_tool_use` 用 `json.dumps` 序列化）。
- 结果：`res.diff` 存在 → `Syntax(diff, "diff", ...)`；否则 `.output/.error` 非空 → `Markdown`；皆空 → `Text("(空结果)")`。
- `ToolBlock` 继承 `Collapsible`，`compose()` 产出参数 `Static(Syntax)`，结果由 `set_result(..., diff)` 在块已挂载后再 `mount`（**不能**在 `__init__` 里 `mount`，否则 `MountError`）。

### ④ 与 `ToolResult.diff` 的接口契约

- `ToolResult(ok, output, error, diff)`：`transport.on_event` 的 `TOOL_RESULT` 分支调 `app.update_tool_result(tc, ev.tool_result)`，
  `app` 透传 `res.diff` 给 `ToolBlock.set_result(result_text, ok, diff)`。
- 高亮依赖上游（写/编辑工具 / sandbox 执行）产生 `res.diff`；本步只负责渲染，不生成 diff。

### ⑤ 两个 Textual 踩坑（M8.4 实测）

1. **不要覆盖 `Static._render()`**：`Static` 内部用 `self._render()` 做布局高度测量。自定义渲染方法必须改名
   （这里用 `_build`），否则 `run_test` 布局阶段抛 `TypeError: _render() missing 1 required positional argument`。
2. **`Collapsible` 的子部件用 `compose()` 声明，勿在 `__init__` 调 `self.mount`**：未挂载到父节点前 `mount` 会抛 `MountError`；
   延迟挂载（如结果区）需等自身已挂载后再调 `self.mount`。
3. **rich `Syntax` 无 `lexer_name` 属性**：本版本（rich 15）`Syntax` 只暴露 `_lexer` / `lexer`，存储源码的属性是 `.code`；
   测试断言 diff 用 `isinstance(x, Syntax)` + `"print(1)" in x.code` 而非 `.lexer_name`。
