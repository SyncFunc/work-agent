## 知识沉淀

- ① **`_SubAgentTuiTransport` vs `_SubAgentTransport`**：
  - 新建 `agent/runtime/_subagent_tui_transport.py`，继承 `TextualTransport`（对应旧版继承 `TerminalTransport`）。
  - **对齐点**：子 agent 有**独立 EventStream** 不变；HITL 屏蔽/委派父传输（澄清/审批由父代理统一决策）；非 HITL 事件（工具块/计划进度/用户）复用父渲染部件。
  - **差异点**：旧版把事件累积成「带框 Panel」列表 + 父 hub 单一 Live（`SubagentPanelHub`）；TUI 版直接把事件**前缀汇入父 `ChatApp` 主消息区 `#log`**，不复用 hub/Live（Textual 自带流式重渲染）。
  - **接线点**：`agent/subagent.py` 的 `spawn` 按 `isinstance(parent_transport, TextualTransport)` 选择 `_SubAgentTuiTransport`（否则沿用 `_SubAgentTransport`），两者互不污染；旧传输继续保留供 `TerminalTransport` CLI。
- ② **子 agent Renderable 汇入主区约定（前缀）**：
  - `_SubAgentTuiTransport.on_event`：连续 `TEXT` 流**首块**加前缀 `▶ subagent: <name>\n`，后续块续写同一 `AssistantMessage`（coalesce 同父）；遇 `TOOL_USE`/`DECISION`/`USER`/`PLAN_PROGRESS` 重置 `_prefixed`，使下一次文本流作为新消息重新加前缀。
  - 工具调用/结果仍渲染为 `ToolBlock`（`Collapsible`）汇入主区，标题不带前缀（保持与父工具一致）；测试断言主区出现 `▶ subagent` 前缀文本 + 至少一个 `ToolBlock`。
  - HITL 委派：`ask`/`approve`/`confirm_plan` 直接 `await self._parent.<method>()`（父为 `TextualTransport` 或嵌套子传输），子 agent 不自弹模态。
- ③ **真 TTY 验收**：本步以 headless `run_test` 完成 `test_subagent_render`（pytest 绿）；真 TTY 体验清单（流式 Markdown / 工具折叠 / diff / HITL / Ctrl+P / 主题 / 鼠标滚动）在 M8.5 已实现、本步新增子 agent 前缀汇入，待真机逐项核对（无阻塞问题）。
- **不变量**：子 agent 独立 EventStream 不变；旧 `_SubAgentTransport` 保留；`AgentTransport` 协议未改。
- 测试：`tests/unit/test_tui.py::test_subagent_render`（两段文本+工具调用+新段文本+DECISION → 主区含 `▶ subagent: researcher` 前缀、且出现 `ToolBlock`）。
