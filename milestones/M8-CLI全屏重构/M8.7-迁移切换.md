# Step M8.7 迁移切换（调研 §7 Phase 3）

## 实现方案

- 目标：迁移切换 —— 默认 `chat` 切到 Textual，`--legacy` 保留旧 `TerminalTransport`；
  稳定后删除旧实现（或保留 `client` 旧路径按需）。
- 改动文件：
  - `agent/cli.py`：`chat` 默认走 Textual（去掉 `--tui` 或默认 `True`）；加 `--legacy` flag
    走旧 `TerminalTransport` / `_chat_repl`。
  - 观察期后（多轮稳定）：删除 `TerminalTransport` 旧渲染、`_chat_repl`、`client` 旧渲染路径
    （或同步切 Textual）；`run`（非交互 / `CliRunner`）始终走非 TUI 录制 / 打印路径。
  - `README.md` / `CODEBUDDY.md` 常用命令更新默认行为。
- 关键接口/算法：无新接口，纯切换；`AgentTransport` 协议不变。
- 依赖/环境：M8.0–M8.6 全部验收通过、真 TTY 体验稳定。

## 验收标准

- [ ] `python -m agent.cli chat`（无 flag）默认进入 Textual TUI；`chat --legacy` 走旧渲染。
- [ ] 全量 `pytest -q` 仍绿（旧 `CliRunner` 单测不动）。
- [ ] 文档（`README`、`CODEBUDDY.md` 常用命令）更新 `chat` 默认行为。
- [ ] 不变量：删除旧实现前 `--legacy` 必须可用；`run` / `client` 行为按需（`client` 可后续切，本期可选）。

## 知识沉淀

- ① **默认切换的 cli 改动点**（`agent/cli.py` 的 `chat` 命令）：
  - 移除 `--tui` 选项，改为 `--legacy`（`typer.Option(False, "--legacy")`）。
  - 新增决策函数 `_chat_should_use_tui(legacy) -> bool` 返回 `(not legacy) and sys.stdin.isatty()`；`chat` 用 `use_tui = _chat_should_use_tui(legacy)`。
  - `use_tui=True` → 构造 `ChatApp(session=..., settings=..., session_store=...)` 并 `app.run()`（Textual 全屏）；否则走旧 `TerminalTransport(interactive=True)` + `_chat_repl` 路径。
  - **非交互自动退回**：非 TTY（CI / 管道 / CliRunner）`isatty()` 为 False → `use_tui=False` → 旧渲染，保证 Textual 不卡死且旧 `CliRunner` 单测继续绿。
  - `run` / `client` 本期**不动**（保持 `TerminalTransport` / 终端客户端渲染）；`client` 全屏化留待后续。
- ② **删除旧实现清单**（本期**未删**，留观察期）：`TerminalTransport` / `_chat_repl` / `client` 旧渲染路径完整保留；`--legacy` 是回退开关。删除时机=多轮真机稳定后，届时同步清理并确认 `run` / `client` 行为。
- ③ **迁移观察期与回滚条件**：TUI 为默认后，若真机出现严重问题，用户可立即 `--legacy` 回退（零风险）；CI 永远走旧渲染（非 TTY）。观察指标=真 TTY 体验清单（流式 Markdown / 工具折叠 / diff / HITL / Ctrl+P / 主题 / 鼠标滚动 / 子 agent 前缀）稳定无阻塞。
- ④ **`run` 始终非 TUI 的约束**：`run` 命令保持 `TerminalTransport(interactive=sys.stdin.isatty())` 单次 `session.step` 录制/打印路径，不受 `chat` 默认切换影响（符合「`run` 始终非交互/非 TUI」约束）。
- **测试**：`tests/unit/test_cli.py` 新增 `test_chat_default_enters_tui_when_tty`（mock `_chat_should_use_tui→True`，断言 `ChatApp.run` 被调用；不传 input 避免真 TUI）+ `test_chat_legacy_flag_uses_old_repl`（断言 TUI 未被使用、旧 REPL 走通）；旧 `test_chat_*` 全保持（非 TTY → 旧渲染）。
- **文档更新**：`README.md`（快速开始 + `chat` 命令说明）、`CODEBUDDY.MD`（常用命令）同步 `chat` 默认 TUI 与 `chat --legacy`。
