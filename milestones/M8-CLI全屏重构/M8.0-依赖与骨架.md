# Step M8.0 依赖与骨架

## 实现方案

- 目标：引入 Textual 依赖并搭出可 headless 启动的空 `ChatApp` 骨架；命令行 `chat --tui` 能启动，
  无交互时安全退出（不崩、不卡 TTY）。旧 `run`/`chat`/`client` 一行不动。
- 改动文件：
  - `pyproject.toml`：新增 `[project.optional-dependencies]` 的 `tui` extra，`textual>=0.80`
    （取当时稳定版，落地时再核对 PyPI）；`dev` extra 也并入 `tui` 以便 CI 跑 `test_tui`。
  - 新建 `agent/tui/__init__.py` 与 `agent/tui/app.py`：`class ChatApp(App)` 最小骨架
    （空 `RichLog` 主区 + `Footer` + `Header`），`CSS` 先留最小布局或留空。
  - 新建 `tests/unit/test_tui.py`：`test_app_boots` 用
    `async with ChatApp().run_test() as pilot:` 断言 `pilot.app` 已 mounted、无异常。
  - `agent/cli.py`：在 `chat` 命令加 `--tui` flag；`True` 时构造 `ChatApp().run()`，
    否则走旧 `TerminalTransport` 路径。**`run`/`client` 子命令永远不进 Textual。**
- 关键接口/算法：
  - `ChatApp(App)`：`async def on_mount` 仅 `self.mount(Header(), RichLog(id="main"), Footer())`；
    `COMMAND_PALETTE`/`BINDINGS` 先留空。
  - cli：`def chat(tui: bool = typer.Option(False, "--tui")):` —— 旧逻辑包在 `if not tui:`；
    `if tui: ChatApp().run()`。
- 依赖/环境：沿用现有 stdout UTF-8 强转（Windows 终端）；CI 用 `run_test` headless 不依赖真 TTY。

## 验收标准

- [ ] `pytest tests/unit/test_tui.py::test_app_boots` 通过（headless，无真 TTY）。
- [ ] `python -m agent.cli chat --tui` 在真 TTY 下能启动并安全退出（无异常、不阻塞）。
- [ ] 不变量：旧 `chat`(无 flag) / `run` / `client` 行为完全不变；`AgentTransport`/`EventStream`
      未改；`textual` 仅在 `tui` extra，未进入核心依赖。

## 知识沉淀

- **依赖**：`textual>=0.80`（实测 8.2.8）放进 `pyproject.toml` 的 `tui` extra，并同步加入 `dev` extra
  使 CI 能跑 `test_tui.py`；安装命令 `pip install -e ".[dev]"` 即含 TUI 栈。核心 `dependencies` 不变。
- **`ChatApp` 骨架挂载结构**：纵向 `Header` / `Vertical(#log RichLog + #input TextArea)` / `Footer`；
  `RichLog(id="log", markup=True, auto_scroll=True)` 是主消息区，`TextArea(id="input")` 是输入区（M8.2 接提交）。
- **`run_test` headless 测试**：`async with ChatApp().run_test() as pilot:` 即可无 TTY 挂载；
  `pytest-asyncio` 已 `asyncio_mode="auto"`，测试用 `async def` 直接 `await`。
- **`chat --tui` 挂载点**：`agent/cli.py` 的 `chat` 命令加 `tui: bool` 选项；在构造 `Session` 之后、
  旧 `TerminalTransport` 之前插入分支——**仅当 `sys.stdin.isatty()` 为真**才 `ChatApp(...).run()`，
  非 TTY 打印提示并 `typer.Exit(1)`，避免 Textual 在管道/CI 下卡死；旧 `chat`（无 flag）/ `run` /
  `client` 路径一行未改。`ChatApp` 的 import 在函数内懒加载，使 `import agent.cli` 不强制依赖 textual。
- **铁律未破**：`AgentTransport` / `EventStream` / `loop` / `daemon` 协议未改；`textual` 仅 dev/tui extra。
