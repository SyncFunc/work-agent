# Step M8.5 主题 + 命令面板 + 状态栏

## 实现方案

- 目标：TCSS 主题（`agent/tui/tui.tcss`）、`CommandPalette`（Ctrl+P 注册 `/plan` 等）、
  `Header` 显示实时 `ctx: NN%`。一并满足旧 `调研-CLI美化方案.md` §3.1 主题诉求（更彻底）。
- 改动文件：
  - 新建 `agent/tui/tui.tcss`：定义布局与配色（暗 / 亮），对齐 Claude Code 观感。
  - `agent/tui/app.py`：`self.theme = settings.ui.theme or "textual-dark"`；
    `COMMAND_PALETTE` 注册 `/plan` / `/exec` / `/skills` / `/compact` / `/context`（绑到 `dispatch_command`）。
  - `Header`：`app.set_interval(1.0, self._refresh_ctx)` 周期读 `context_mgr.estimate_usage()`
    显示 ctx%（经 `app.call_later`），替代旧 `_status_line` 塞 prompt 前缀的 hack。
- 关键接口/算法：
  - `settings.ui.theme` 新增字段（`pydantic-settings`，见 CODEBUDDY.md §5）；映射到 Textual 主题名。
  - CommandPalette 命令 = `Command` 对象，动作 `self.dispatch_command(name)`。
- 依赖/环境：M8.2 的 `dispatch_command`；`context_mgr` 实例（现有）。

## 知识沉淀

- ① **`settings.ui.theme` 字段**：`agent/config/settings.py` 新增 `UIConfig(BaseModel)`（`theme: str = "textual-dark"`），主 `Settings` 加 `ui: UIConfig = UIConfig()`。
  - 仅影响 TUI 外观，不改任何 core / TerminalTransport。默认 `textual-dark`（对齐 Claude Code 暗色）。
  - 取值为 Textual 已注册主题名：`textual-dark` / `textual-light` / `catppuccin-mocha` / `catppuccin-latte` / `dracula` / `github-dark` / `github-light` / `gruvbox` / `monokai` / `solarized-dark` / `solarized-light` / `nord` / `flexoki-dark` / `flexoki-light`（见 `BUILTIN_THEMES`）。
  - 辅助 `ui_theme(name)` 规整：空值落回 `"textual-dark"`；非法名 Textual 会回退默认主题不报错（代码以 try/except 兜底）。
  - 应用点：`ChatApp.on_mount` 里 `self.theme = ui_theme(self.settings.ui.theme if self.settings else None)`。
- ② **`tui.tcss` 主题文件**：`agent/tui/tui.tcss`（`CSS_PATH = [Path(__file__).parent / "tui.tcss"]`）。
  - 关键布局：`Screen { background: $surface }`、`Header { height:1 }`、`#log { height:1fr }`、`#input { height:5; border:round $primary }` 聚焦时 `$accent`、`Footer`、`ToolBlock { border:round $panel }`。
  - **踩坑**：`text-style` 合法值不含 `normal`，要用 `none`；否则 `StylesheetParseError: Invalid value 'normal' in text-style property`（整屏 compose 直接崩，所有 TUI 测试失败）。
  - 类型选择器可用下划线类名（`_StaticLine`），但 `ToolBlock.-collapsed` 这种组合需谨慎（本次仅留 `height:auto`）。
- ③ **`CommandPalette` 命令注册范式**（Textual 8.2.8，无内置 `COMMANDS`）：
  - 模块在 `textual.command`：`CommandPalette`、`Provider`、`Hit`、`Hits`。
  - `Provider` 基类 `__init__(self, screen, match_style=None)`；需实现 `async def search(self, query) -> Hits`（yield `Hit(score, match_display, command, text, help)`）与可选 `discover()`。
  - `Hit` 构造：`Hit(1.0, name, self._make_run(name), name, desc)` —— `match_display`/`text` 用命令名字符串，`command` 是无参可调用。
  - **注册方式**：不靠 `App.COMMANDS`，而是在 `BINDINGS` 加 `("ctrl+p", "open_commands", "命令面板")`，并实现 `action_open_commands` → `self.push_screen(CommandPalette(providers=[AgentCommandProvider]))`。避免默认 `action_command_palette` 因无 providers 显示空面板。
  - `Provider.app` 是 property（返回 `self._screen.app`）；`command` 回调经 `self.app.call_later(self.app._handle_command, name)` 回主线程。`call_later` **会 await 协程**（已实测），故 async `_handle_command` 能正常执行。
- ④ **`Header` ctx% 刷新机制**：
  - **关键**：`sub_title` 是 **`App`** 的属性（`Header` 自动渲染），**不是 `Header.sub_title`**！`self.query_one(Header).sub_title` 会 `AttributeError`。直接设 `self.sub_title = pct` 即可。
  - `on_mount` 里 `self.set_interval(1.0, self._refresh_ctx)`，`_refresh_ctx` 读 `self.session.context_mgr.estimate_usage().used_pct`（0.0–1.0），格式 `ctx: NN%`，仅在变化时才赋值（避免每帧重渲染）。
  - `session` 为 None 或无 `context_mgr` 时 `pct=""`，状态栏清空；全程包 try/except 防单次统计异常崩 UI。
- **测试**：`test_theme_switch`（默认 `textual-dark`、切 `catppuccin-mocha` 生效）、`test_command_palette`（Ctrl+P 推 `CommandPalette` + `AgentCommandProvider.search("/compact")` 命中 + 回调经 `call_later` 调 `_handle_command("/compact")`）、`test_ctx_header`（注入 mock `context_mgr`，断言 `app.sub_title` 含 `ctx:` 与百分比，且随 `used_pct` 变化）。
- **不变量**：`settings.ui.theme` 默认不影响旧 CLI；旧 `调研-CLI美化方案.md` 主题诉求被本步彻底满足（TCSS 暗色主题 + 可切主题）。
