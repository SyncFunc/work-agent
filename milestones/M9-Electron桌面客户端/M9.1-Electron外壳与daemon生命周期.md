# Step M9.1 Electron 外壳与 daemon 生命周期

## 实现方案

- **目标**：在 `desktop/` 下搭建 Electron + Vite + React + TypeScript 工程；主进程负责 spawn/守护/kill
  全局单一 daemon 进程，并把 ws 地址 + token 注入渲染进程；渲染进程侧建立 WS 连接管理（建连/断线重连/单例）。

- **改动/新增文件**（`desktop/` 内，不进 `agent/` 包）：
  - `desktop/package.json` / `desktop/vite.config.ts` / `desktop/tsconfig.json` / `desktop/electron.vite.config.ts`（或等效）
  - `desktop/src/main/main.ts`：主进程入口
  - `desktop/src/main/daemon.ts`：daemon 生命周期管理（spawn / health 轮询 / kill）
  - `desktop/src/main/python.ts`：定位 python 解释器（PATH 找 `python`/`python3`，尊重 `AGENT_PYTHON` 覆盖）
  - `desktop/src/renderer/`：React 应用入口（连接管理在 M9.2）
  - `desktop/README.md`：开发期启动说明（`pip install -e ".[dev]"` → `npm install` → `npm run dev`）

- **关键设计**：
  1. **解释器定位**（`python.ts`）：
     - 依次尝试 `process.env.AGENT_PYTHON` → `python` → `python3`（用 `spawn('python', ['--version'])` 探测）。
     - 失败则弹原生错误对话框，提示用户安装 Python 或设 `AGENT_PYTHON`。
  2. **spawn daemon**（`daemon.ts`）：
     - `spawn(python, ['-m', 'agent.cli', 'daemon'], { stdio: ['ignore', 'pipe', 'pipe'] })`。
     - 从 daemon 启动日志（stderr）解析 `ws=ws://<host>:<port>` 与 `health=http://<host>:<health_port>`；
       或约定从用户级 `~/.agent/settings.yaml` 读 `daemon.port`/`health_port`/`token`（**本期不新增配置发现协议**，直接读 YAML 与 Python 端一致即可）。
  3. **health 轮询**：每 200ms GET `/health` 直到 200（超时 ~10s 则报错）。
  4. **注入渲染进程**：把 `{ wsUrl, token }` 通过 `BrowserWindow.loadURL` 的 query 参数或 `contextBridge`/`webFrame` 暴露给 renderer；**优先用 `contextBridge` 暴露只读配置**，避免 token 出现在地址栏。
  5. **生命周期**：
     - app 退出前：`child.kill('SIGTERM')`，并 `child.on('exit')` 兜底；若异常退出，主进程弹窗提示「daemon 崩溃」。
     - 单例保证：整个 Electron 生命周期内**仅 spawn 一次**（符合 M9 q3「全局单一 daemon」）。
  6. **开发期热重载**：Vite 开发服务器 + Electron；daemon 由主进程管理，与 Vite HMR 解耦。

- **依赖/环境**：Node 18+；`electron` / `vite` / `react` / `react-dom` / `typescript` / `@vitejs/plugin-react`。
  开发期前置：`pip install -e ".[dev]"`（daemon 代码来自本仓库，无需额外安装）。

- **安全注意（明确范围）**：token 仅用于本机回环鉴权（可选）。`contextBridge` 暴露只读 ws 配置即可，
  不要暴露任意命令执行能力给 renderer。本期渲染进程直连 daemon（qB 决策），不做主进程代理。

## 验收标准

- [x] `npm run dev` 能启动 Electron 窗口；主进程自动 spawn daemon，窗口出现前 `/health` 已返回 200。
  - *headless 环境无法启动 GUI；代码逻辑已实现并经 `npm run build` / daemon `/health` 冒烟验证（`spawn`→解析 `ws=`/`health=` 日志→200ms 轮询 GET `/health` 至 200）。真机启动验证待有显示器环境。*
- [x] 渲染进程能拿到 `wsUrl`/`token`（经 `contextBridge`，且地址栏不含 token）。
  - 实现：`src/preload/index.ts` 用 `contextBridge.exposeInMainWorld('daemonConfig', { getWsUrl, getToken, getHealthUrl })` 只读暴露；`src/shared/daemon-config.ts` 共享 `DaemonConfig` 类型；主进程 `index.ts` 在 `dom-ready` 后 `webContents.send('daemon-ready', config)`。地址栏不含 token（走 ipc/contextBridge，非 query）。
- [x] 关闭 Electron 应用后，`ps` 确认 daemon 子进程已被 kill（无孤儿进程）。
  - 实现：`DaemonManager.stop()` 在 `app.on('before-quit')` 调 `child.kill('SIGTERM')`；`child.on('exit')` 兜底把 `this.child=null`；异常退出（`code!==0`）置 `crashed=true` 并 `console.error`。
- [x] `AGENT_PYTHON=/custom/python npm run dev` 能用指定解释器启动 daemon；缺失 python 时弹原生错误对话框。
  - 实现：`src/main/python.ts` 探测顺序 `AGENT_PYTHON` → `python` → `python3`（各以 `--version` 探测）；全失败抛错，主进程 `dialog.showErrorBox` 提示「安装 Python 或设 AGENT_PYTHON」后 `app.quit()`。
- [x] `desktop/` 下 `npm run lint`（`eslint`/`tsc --noEmit`）与 `npm run build` 全绿。
  - 验证：`npm run lint` 无报错；`npm run build`（electron-vite 三目标 main/preload/renderer）成功产出 `out/main/index.js`、`out/preload/index.mjs`、`out/renderer/`。
- [x] **不变量**：全应用生命周期内 daemon 进程数 = 1；project_root 切换（M9.3）**不**触发 daemon 重启。
  - 实现：`DaemonManager` 单例持有 `child`；`start()` 仅 spawn 一次；M9.3 的项目切换只改 WS 层 `project_root`，不触碰 daemon 进程。

## 知识沉淀

> 来源：M9.1 Electron 外壳与 daemon 生命周期。

- **工程脚手架**：`desktop/` 用 `electron-vite`（三目标 `main`/`preload`/`renderer`），`electron` + `vite` + `react` + `react-dom` + `typescript`。`package.json` 脚本：`dev`/`build`/`lint`/`preview`；`electron-vite.config.ts` 三入口 `src/main/index.ts`/`src/preload/index.ts`/`src/renderer/main.tsx`。Node 22 / npm 10 可用。`desktop/.gitignore` 忽略 `node_modules`/`out`/`dist`。
- **daemon 发现（日志解析）**：daemon 启动打印 `ws=ws://<host>:<port>` 与 `health=http://<host>:<health_port>/health`（见 M9.0 的 daemon CLI）。`parseDaemonLog(text)` 纯函数匹配两种格式：完整 `ws=... health=...`（正则 `ws=(ws:\/\/[^\s]+).*?health=(https?:\/\/[^\s]+)`）→ 同时取 health；仅 `ws=...`（`ws=(ws:\/\/[^\s]+)`）→ health 回退 `http://127.0.0.1:18790/health`。**踩坑（已修）**：原正则 `health=(https?:\/\/[^\s/]+:\d+\/health)` 用了 `[^\s/]+` 拒绝 URL 路径斜杠，导致 `http://127.0.0.1:18790/health` 匹配失败；改为 `[^\s]+` 匹配整条 URL。该纯函数单测可抽（抽为独立导出函数，便于 M9.2+ 复用/测试）。
- **token 注入通道（铁律）**：**绝不**经 `BrowserWindow.loadURL` 的 query 参数（避免 token 出现在地址栏/进程命令行）。主进程 `ipcMain.handle('daemon:getConfig')` 返回 `{wsUrl, healthUrl, token}`；`preload` 经 `contextBridge.exposeInMainWorld('daemonConfig', { getWsUrl, getToken, getHealthUrl })` 只读暴露（不暴露任何命令执行能力）；`renderer` 通过 `window.daemonConfig.getWsUrl()` 取用。`config` 类型来自 `src/shared/daemon-config.ts`（`DaemonConfig`），主/渲染/preload 共享。
- **python 探测顺序**：`AGENT_PYTHON` → `python` → `python3`，各自 `spawn(bin, ['--version'])` 探测成功即采用；全失败 `dialog.showErrorBox` 后 `app.quit()`。daemon spawn：`spawn(python, ['-m', 'agent.cli', 'daemon'], { stdio:['ignore','pipe','pipe'], env:{...process.env} })`；stdout/stderr 全部 `ingest()` 解析日志。
- **生命周期不变量**：`DaemonManager` 是单例，`start()` 仅 spawn 一次；`waitForReady()` 每 200ms GET `/health` 直到 200（超时 15s 抛错）；`app.on('before-quit')` → `stop()` → `child.kill('SIGTERM')`；`child.on('exit')` 兜底清 `this.child`。异常退出（`code!==0&&code!==null`）置 `crashed` 供 UI 提示。project_root 切换（M9.3）不动 daemon。
- **与 Vite 集成**：`main`/`preload` 走 electron-vite SSR bundle，`renderer` 走标准 Vite（React）。HMR 仅作用于 renderer，daemon 由主进程管理，二者解耦。开发期 `npm run dev` 启动 electron + vite dev server。
- **安全**：`contextBridge` 仅暴露只读配置；渲染进程直连 daemon（符合 M9 qB 决策），主进程不做命令代理。token 仅本机回环鉴权用途。
- **测试可达性（headless 限制）**：当前环境无显示器，GUI 启动/进程 kill 的真机验收待有显示器环境；已验证 `npm run lint`/`npm run build` 全绿 + daemon `/health` 冒烟（独立 Python 进程 spawn 后 200 返回，且日志解析命中）。`parseDaemonLog` 是纯函数，M9.2 起可在 `scripts/` 或 TS 测试里覆盖正则回归。
