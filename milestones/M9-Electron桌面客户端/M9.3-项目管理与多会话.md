# Step M9.3 项目管理与多会话

## 实现方案

- **目标**：实现「项目根切换器 + 多会话标签页 + 会话列表 + resume/fork + 重放」，对齐 M7.4 的 `session.list`/`attach`/`switch`/`replay_start|end` 与 M7.5 的 `/resume` `/fork` 命令。

- **新增文件**（`desktop/src/`）：
  - `features/projects/ProjectSwitcher.tsx`：左侧项目根切换（输入/浏览选择目录；当前 `project_root` 持久化到 `localStorage`）。
  - `features/sessions/SessionTabs.tsx`：多会话标签页（每个 tab = 一个 `session_id` + `project_root`）。
  - `features/sessions/SessionList.tsx`：当前项目下会话列表（`session.list` → `session_list`）。
  - `features/sessions/useSessions.ts`：会话状态机（连接 `DaemonClient`，管理 `new/attach/switch/detach`、`replay` 缓冲）。
  - `features/sessions/ReplayBuffer.ts`：消费 `replay_start`→批量 `event`→`replay_end`，重建历史渲染（瞬时 `tool_call_delta` 不入 replay，与 daemon 一致）。

- **关键设计**：
  1. **项目切换**：切换 `project_root` → 调 `listSessions(projectRoot)` 刷新列表；当前激活会话随之切换（同一 daemon，不重启）。
  2. **多标签页**：每个 tab 记录 `{session_id, project_root}`；新会话 `newSession(name, projectRoot)` 开 tab 并 `attach`；
     切换 tab = `switch(session_id, projectRoot)`（触发 replay 回填历史）。
  3. **resume / fork**：
     - resume：`session.list` 选中历史会话 → `attach(session_id, projectRoot)`。
     - fork：调 `command('fork', session_id)`（M7.5 透传 `dispatch_command`）；若 M9.0 已实现 `command_result` 回传新 `session_id` 则直接开新 tab，否则 fork 后重新 `listSessions` 取最新并 `attach`。
  4. **重放**：`attach/switch` 后按 `replay_start/end` 区间把历史 `Event` 灌入渲染（M9.4 消费），保持瞬时事件排除规则。
  5. **空状态**：当前项目无会话时展示「新建会话」入口。

- **依赖/环境**：M9.0 daemon（携带 `project_root`）、M9.2 `DaemonClient`。

## 验收标准

- [ ] 在 A 项目新建 2 个会话、B 项目新建 1 个会话；切换项目根时列表与标签页只显示对应项目会话（隔离由 M9.0 保证）。
- [ ] `attach`/`switch` 后历史消息经 replay 完整回填，且 `tool_call_delta` 等瞬时事件不重复渲染。
- [ ] resume 能载入历史会话并继续对话；fork 产出新会话 tab 且内容独立。
- [ ] 关闭某 tab（detach）后 daemon 侧该连接释放，会话仍在（可再次 attach）。
- [ ] `npm run lint` / `tsc --noEmit` 全绿。
