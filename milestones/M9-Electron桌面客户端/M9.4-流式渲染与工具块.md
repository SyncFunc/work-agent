# Step M9.4 流式渲染与工具块

## 实现方案

- **目标**：把 daemon 推来的 `Event` 流渲染为「流式 Markdown + 可折叠工具块（diff 高亮）+ 参数/结果分离」，
  等价于 M8 Textual TUI 的渲染层（代码不复用，仅借鉴交互设计）。

- **新增文件**（`desktop/src/features/chat/`）：
  - `MessageList.tsx` / `MessageItem.tsx`：按 `seq` 顺序渲染事件。
  - `Markdown.tsx`：流式 Markdown（支持增量追加，`react-markdown` + `remark-gfm` + 代码高亮 `rehype-highlight`）。
  - `ToolBlock.tsx`：工具调用卡（默认折叠/展开）；`tool_use` → 标题+参数；`tool_call_delta` → 实时参数预览；`tool_result` → 结果（可折叠，超长截断+展开）。
  - `DiffView.tsx`：从 `tool_result.output` 中识别 diff（以 `---`/`+++`/`@@` 或 unified diff 特征）并语法高亮（diff 语法）。
  - `useEventReducer.ts`：把 `Event[]` 归约为「消息/工具块」视图模型（按 `seq`，流式合并 `text` 增量与 `tool_call_delta`）。

- **关键设计**（对齐 `agent/core/events.py` `EventType`）：
  1. **流式文本**：`TEXT`（`kind=reasoning|content`）与 `DECISION.text` 增量追加到对应消息气泡；`reasoning` 与 `content` 分栏/折叠显示（思考过程可收起）。
  2. **工具块**：
     - `TOOL_USE{tool_call}` → 新建工具块（名称 + 参数 JSON 预览 + 风险标记）。
     - `TOOL_CALL_DELTA{tc_index,tc_name,tc_args}` → 实时更新对应工具块参数预览（瞬时，不入 replay，与 daemon 一致）。
     - `TOOL_RESULT{ok,output,error}` → 填入结果区；失败标红；超长 `output` 截断 + 「展开」。
  3. **参数/结果分离**：同一工具块内 `参数`（只读 JSON）与 `结果`（可 diff 高亮）分区，便于审计。
  4. **diff 高亮**：`tool_result.output` 命中 diff 模式 → `DiffView` 着色（增绿/删红/上下文灰）；非 diff 普通代码高亮。
  5. **`FINAL` / `ERROR` / `USER`**：FINAL 收尾消息；ERROR 红色告警；USER 为用户侧输入气泡（来自 `Event.type==user`）。
  6. **顺序正确性**：依赖 `seq` 严格保序；`replay` 与实时事件走同一 reducer，避免重复/错位。

- **依赖/环境**：`react-markdown` `remark-gfm` `rehype-highlight`（`highlight.js` 主题）；M9.2 `parseEvent` 提供数据。

## 知识沉淀

### Event→视图模型 reducer 规则（`features/chat/useEventReducer.ts`）
- `buildChatModel(events)` 纯函数：`TEXT` 增量累积进「当前 assistant 文本气泡」的 `content`/`reasoning`；`DECISION`/`FINAL` 仅作**收尾信号**——流式场景下 `.text` 与已追加 `TEXT` 重复，故**仅在当前气泡无任何 TEXT 增量时**用 `decision.text`/`final.text` 兜底（非流式兼容）。这与 `agent/runtime/textual_transport.py` `on_event` 的 `finalize_stream` 语义一致。
- 工具块三阶段：`TOOL_CALL_DELTA{tc_index,tc_args}`（瞬时，在 `TOOL_USE` **之前**到达，按 `tc_index` 累积进 `deltaArgs` 预览）→ `TOOL_USE{tool_use}`（补齐真实 `id`/最终 `args`，按出现序匹配 delta 创建的块）→ `TOOL_RESULT{tool_call_id,tool_result}`（填结果、`running=false`）。
- **顺序正确性**：`TOOL_CALL_DELTA`/`TOOL_USE` 到达时先 `flushText()`，确保文本气泡先于工具块落位（否则工具预览会跑到文本前面）。replay 与实时走同一 reducer。
- 块类型：`text`/`tool`/`user`/`error`/`clarify`/`plan`；React `key` 在块创建时一次性分配并复用（delta→use→result 全程同一对象引用，原地 mutate 即更新渲染）。

### diff 识别（`features/chat/DiffView.tsx`）
- `isDiffLike`：`/^[+-]\s|@@ |diff --git |index |\+\+\+ |--- /` 命中 ≥3 行判为 unified diff；逐行着色（增绿 `wa-diff-add` / 删红 `wa-diff-del` / hunk `wa-diff-hunk` / 上下文 `wa-diff-ctx`）。非 diff 走 `rehype-highlight` 普通代码高亮。

### Markdown 增量渲染策略
- 不逐字符 diff，整段 `content` 交给 `react-markdown` 重渲染；`reasoning`/`content` 分栏，思考过程可折叠。依赖 `highlight.js/styles/github.css` 主题（在 `Markdown.tsx` 内 `import`）。

- 新增 `react-markdown` `remark-gfm` `rehype-highlight` `highlight.js`（desktop deps）；`npm run build` 产物含高亮样式 `assets/index-*.css`。
