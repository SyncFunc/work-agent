# Step M9.6 设置面板与命令面板

## 实现方案

- **目标**：① 设置面板编辑 LLM/计划/主题等配置（写项目级 `settings.yaml`，与 `agent/config/settings.py` 字段一致）；
  ② 命令面板 + 斜杠命令，触发 M7.5 的 `command` 消息（`/context` `/compact` `/plan` `/skills` `/agents` `/mode` 等）。

- **新增文件**（`desktop/src/features/`）：
  - `settings/SettingsPanel.tsx`：表单编辑 `LLMConfig`(api_key/base_url/model) / `PlanConfig`(mode) / `ClarifyConfig`(enabled) / `UIConfig`(theme) / `SandboxConfig`(profile) / `ApprovalConfig`(mode)。
  - `settings/settingsApi.ts`：读/写项目级 `<project_root>/.agent/settings.yaml`（Electron `fs` + `js-yaml`）；
    写入后提示「重启会话/daemon 生效」（本期不做热重载，仅持久化）。
  - `command/CommandPalette.tsx`：Ctrl/Cmd+K 唤起；搜索并执行斜杠命令。
  - `command/useCommands.ts`：命令注册表（映射 M7.5 `_HELP_ROWS`：/plan /exec /approve /mode /context /compact /skills /agents /skill /agent /bg /sessions /resume /fork /help）。
  - `chat/InputBar.tsx`：输入框支持 `/` 前缀直接发 `command(name, args)`（与 M7.3 CLI REPL 一致）。

- **关键设计**：
  1. **设置写回**：表单值 → YAML 合并写入 `<project_root>/.agent/settings.yaml`（仅覆盖用户改的键，保留其他配置）。
     - 注意：daemon 按 `project_root` 惰性 `load_settings`，但**已存在的会话沿用创建时的 settings**；设置生效范围在文档中明确（新会话/重启 daemon 后全生效）。
     - `api_key` 敏感字段：输入框 `type=password`，本地不展示明文（仍明文落盘 YAML，与现有约定一致；后续打包可加系统密钥库）。
  2. **命令面板**：选中命令 → 构造 `/name args` → `DaemonClient.command(name, args)`（daemon 经 `dispatch_command` 处理并 `notify` 反馈，前端订阅 `notify` 展示结果）。
  3. **斜杠命令直输**：`InputBar` 以 `/` 开头 → 解析 `name`/`args` 发 `command`（复用 M7.3 `_parse_command` 语义）。
  4. **theme**：写 `UIConfig.theme`，本地同时切换渲染进程 CSS 主题（与 Textual 主题名解耦——前端用自有主题名，仅持久化字段对齐）。

- **依赖/环境**：`js-yaml`；M9.2 `DaemonClient.command`；设置面板依赖 M9.3 的当前 `project_root`。

## 知识沉淀

### settings 写回合并策略（`main/settings.ts` + `preload` + `features/settings`）
- 渲染进程**无 node fs**（Electron `contextIsolation`+`nodeIntegration:false`），故设置读写经 IPC：主进程 `ipcMain.handle('settings:read'/'settings:write')` ↔ preload `agentApi.readSettings/writeSettings` ↔ 渲染 `settingsApi.loadSettings/saveSettings`。
- `mergeSettings(base, patch)` 深合并（对象递归、数组整体替换），仅覆盖 patch 中的键，保留其余配置；`js-yaml` v4 用**具名导入** `load`/`dump`（无 default 导出，否则 rollup 报 “default is not exported”）。
- 生效范围：daemon 惰性 `load_settings(project_root)`；**已创建会话沿用创建时 settings**，故设置对新建会话/重启 daemon 生效——UI 保存后提示「新会话/重启 daemon 生效」。

### 命令面板与 M7.5 命令清单同步
- `useCommands` 的 `COMMANDS` 静态对齐 M7.5 命令集（`/context` `/compact` `/plan` `/skills` `/agents` `/mode` `/exec` `/approve` `/bg` `/sessions` `/resume` `/fork` `/skill` `/agent` `/help`）；`resume`/`fork` 标 `needsSession`。
- 执行 → `DaemonClient.command(name, args)`；daemon `dispatch_command` 经 `notify`/`show_skills`/`show_agents` 反馈（前端 `useNotices` 订阅展示 toast）。`InputBar` 以 `/` 前缀走 `parseSlash` 直发 `command`（与 M7.3 CLI REPL 一致）。

### theme 前后端映射
- 前端自有主题名 `light`/`dark`，持久化到 `localStorage`（即时生效）并同步写入 `ui.theme`（与 `settings.py` 字段对齐）；`theme.css` 响应 `body[data-theme="dark"]`（内联样式为主，仅全局元素主题化，证明机制）。
