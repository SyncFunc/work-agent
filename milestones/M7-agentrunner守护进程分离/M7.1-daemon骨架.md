# Step M7.1 daemon 骨架（SessionRegistry + WS / HTTP Server）

## 实现方案
- **目标**：daemon 常驻、多 `Session` 索引、按连接路由消息、每会话并发 step 串行化。
- **改动文件（新增）**：`agent/daemon/__init__.py`、`agent/daemon/registry.py`、`agent/daemon/server.py`；`agent/cli.py` 新增 `daemon` 子命令。
- **关键结构**：
  - `SessionRegistry`：`new(name?)->SessionHandle`、`attach(conn, session_id)`、`detach(conn)`、`switch(conn, session_id)`、`list()`、`get(session_id)`。
  - `SessionHandle`：持有 `Session`、环形缓冲 `event_buffer: deque(maxlen=K)`（**仅收非 `transient` 事件**，见全局约定修复点②）、`attached_conn`、`running: bool`、`last_activity`、每会话 `asyncio.Lock`（保证同一会话一次仅一个 step 在飞）。
  - `server.py`：`websockets.serve(handler, host="127.0.0.1", port=settings.daemon.port)`，默认 `18789`；`handler(conn)` 按消息 `type` 路由；管理每连接「当前 attached session」。每个 `task.send` 触发一次 `Session.step`，用**每会话 `asyncio.Lock`** 保证同一会话一次只有一个 step 在飞（并发 `task.send` 排队或回 `error` code=busy）。
  - `Session` 构造复用 `cli.py` 现有逻辑（load_settings → model → TraceStore → Session），需 `Session` 暴露只读 `event_stream` 供 `bind`（允许 core 极小扩展）。
- **依赖 / 环境**：`websockets`（仅本路径 import）；daemon 仅绑 `127.0.0.1`；SIGINT 优雅关闭（等待 `Session.shutdown_background`）。

## 验收标准
- [ ] 启动 `agent daemon` 后 HTTP `/health` 返回 200；`hello` 后回 `welcome`（含 `daemon_version` / `protocol_version`）。
- [ ] `session.new` 创建会话并回 `session.created`；`session.list` 返回清单（含 id / name / attached / running / last_activity）。
- [ ] 同一会话并发 `task.send` 被每会话 `asyncio.Lock` 串行化或回 `error(busy)`，不交叉执行。
- [ ] 单测：`SessionRegistry.new/attach/detach/switch/list` 行为正确；每会话锁生效。
- [ ] **不变量**：`agent/core/{loop,session,transport,events}.py` 无实质改动（仅允许 `Session` 暴露只读 `event_stream`）。

## 知识沉淀
> 设计期：daemon 用单一 asyncio 事件循环驱动所有 `Session`；多会话并发靠**每会话 Lock** 而非全局锁，避免互锁。`SessionHandle.event_buffer` 只收非 `transient` 事件（回放不重画参数预览）。`Session` 暴露只读 `event_stream` 是唯一的 core 扩展点之一。
