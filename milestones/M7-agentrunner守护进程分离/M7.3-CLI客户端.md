# Step M7.3 CLI 客户端（渲染复用 + HITL 回传）

## 实现方案
- **目标**：CLI 客户端连 daemon，复用既有终端渲染，把 HITL 经协议回传。
- **改动文件（新增）**：`agent/daemon/client.py`；`agent/cli.py` 新增 `client` 子命令。
- **客户端行为**：`async` ws 客户端。`hello` 后默认 `session.new`（或按 `--session <id>` attach / `--resume` 最近会话）。REPL：输入命中 `/...` 命令 → 发 `command`；否则发 `task.send`。`--run "<task>"` 一次性模式（发单条 `task.send` 后等待 `close` 即退出，对标 `run`）。
- **渲染复用**：客户端持有 `TerminalTransport(interactive=isatty(), context_mgr=None)`，对每个收到的 `event` 消息执行 `transport._on_event(Event.from_dict(payload["event"]))`——**直接复用既有事件→rich 渲染映射，TerminalTransport 零改动**（其 `__init__` 已初始化全部渲染状态；`bind` 在 daemon 侧由 `BridgeTransport` 完成，客户端不调 `bind`）。
- **HITL 回传**：收到 `ask` → 复用 `TerminalTransport` 内的 `_ptk_single_choice` / `_ptk_multi_choice` 提问 → 发 `answer`{id, text}；`confirm_plan` / `approve` → 复用 `TerminalTransport.confirm_plan` / `approve` 的 prompt 逻辑 → 发对应应答。

## 验收标准
- [x] `agent client` 连接 daemon、`hello` 后默认建会话；`--session <id>` 可 attach 指定会话；`--run` 发单任务后等 `close` 退出。
- [x] 收到 `event` 经 `TerminalTransport._on_event` 渲染，效果与进程内 `chat` 一致（含 `text` 逐字、`tool_call_delta` 参数预览）。
- [x] 收到 `ask` / `confirm_plan` / `approve` 时就地提问并回传同 `id` 应答，daemon 侧 `BridgeTransport` 被唤醒继续 `Session.step`。
- [x] `TerminalTransport` 源码不经本次改动（`git diff` 不含 `agent/runtime/terminal_transport.py` 实质改动）。

## 知识沉淀
> 设计期：客户端是「哑渲染 + 输入」端，所有语义（命令解释、HITL 决策）由远端 `Session` 经协议驱动。复用 `TerminalTransport._on_event` 是零成本复用点，避免重写 rich 渲染映射。
